# 1. Executive Summary

## 1.1 Business Challenge

Для большинства компаний рост клиентской базы сам по себе не гарантирует устойчивого увеличения выручки. Значительная часть экономического результата определяется способностью бизнеса удерживать существующих клиентов, своевременно выявлять признаки снижения активности и эффективно распределять ресурсы на retention-программы.

В рассматриваемой аптечной сети клиенты существенно различаются по уровню вовлеченности, покупательскому поведению и долгосрочной ценности. В этих условиях использование единых CRM-коммуникаций для всей клиентской базы приводит к снижению эффективности маркетинговых инвестиций и затрудняет управление рисками потери выручки.

Основная бизнес-задача проекта заключается в построении комплексной системы клиентской аналитики, позволяющей:

* понимать механизмы формирования клиентской ценности;
* выявлять ключевые источники потерь выручки;
* анализировать динамику удержания клиентов;
* прогнозировать риск оттока;
* оценивать потенциальный объем revenue-at-risk;
* определять приоритетные сегменты для retention-интервенций.

Проект охватывает полный цикл Customer Analytics: от анализа жизненного цикла клиентов и когортной динамики до предиктивного моделирования и разработки системы приоритизации retention-инвестиций.

#

## 1.2 Analytical Approach

Для решения поставленной задачи была разработана многоуровневая аналитическая архитектура, объединяющая четыре взаимосвязанных слоя анализа.

### Layer 1. Lifecycle Transition Analytics

Исследуются переходы клиентов между стадиями жизненного цикла и влияние этих переходов на динамику выручки.

Основной вопрос:

> Как изменение структуры клиентской базы формирует рост или потерю выручки?

В рамках слоя анализируются:

* lifecycle transitions;
* миграция клиентов между сегментами;
* драйверы роста и снижения выручки;
* экономика удержанных клиентов.

#

### Layer 2. Cohort Retention & Customer Value Dynamics

Исследуются долгосрочные закономерности удержания и формирования клиентской ценности.

Основной вопрос:

> Как меняются retention и customer value по мере развития клиентских когорт?

В рамках слоя анализируются:

* user retention;
* revenue retention;
* retention decay;
* customer lifetime value;
* динамика Revenue per Retained User;
* межкогортная вариативность поведения.

#

### Layer 3. Predictive Customer Analytics

Разрабатывается система прогнозирования пользовательского поведения и оценки экономических рисков.

Основной вопрос:

> Какие клиенты наиболее вероятно будут потеряны и какие из них являются наиболее значимыми для бизнеса?

В рамках слоя реализованы:

* churn prediction;
* SHAP-анализ факторов риска;
* revenue-at-risk framework;
* recovery modeling;
* retention prioritization framework.

Для прогнозирования риска оттока использовалась модель Random Forest с временным разделением выборки (time-based split), что позволяет приблизить анализ к реальному бизнес-сценарию использования модели.

#

### Layer 4. Lifecycle KPI Framework & Customer Health Monitoring

Формируется система мониторинга состояния клиентской базы и раннего выявления угроз для выручки.

Основной вопрос:

> Какие показатели необходимо отслеживать для оперативного управления retention-рисками?

В рамках слоя разработан набор KPI, объединяющий показатели удержания, стабильности клиентской базы, концентрации выручки и потенциальных потерь дохода.


## 1.3 Key Findings

### Retention Dynamics

Анализ когорт показал наличие выраженного раннего оттока клиентов.

Средний User Retention снижается до приблизительно 26% уже ко второму месяцу после первой покупки, после чего скорость снижения резко замедляется.

Точка стабилизации retention-кривой формируется около второго месяца жизненного цикла клиента, а долгосрочный уровень удержания остается относительно стабильным в диапазоне 20–24%.

Одновременно Revenue Retention систематически превышает User Retention, что свидетельствует о более высокой экономической ценности удержанных клиентов.

#

### Revenue Drivers

Основным источником роста выручки выступает Expansion внутри существующей клиентской базы.

| Revenue Driver | Revenue Change |
| -------------- | -------------: |
| Expansion      |      +2.74 млн |
| Reactivation   |      +0.36 млн |
| Acquisition    |      +0.05 млн |
| Contraction    |      −1.69 млн |
| Churn          |      −1.51 млн |

Наиболее существенный положительный вклад в динамику выручки обеспечивают клиенты, сохраняющие активность внутри сегмента Loyal.

Ключевым источником потерь выступает переход клиентов из сегмента At Risk в Churn.

#

### Customer Value Evolution

Несмотря на сокращение числа удержанных клиентов, средняя ценность оставшейся аудитории постепенно возрастает.

Revenue per Retained User демонстрирует рост до 43% относительно ранних этапов жизненного цикла, а зрелые когорты формируют средний LTV на уровне 6.3–6.8 тыс. рублей.

Полученные результаты свидетельствуют о возрастающей концентрации долгосрочной выручки среди наиболее лояльных клиентов.

#

### Churn Prediction

Разработанная модель прогнозирования оттока продемонстрировала высокое качество классификации.

| Metric    | Value |
| --------- | ----: |
| ROC-AUC   |  0.89 |
| Precision |  0.76 |
| Recall    |  0.95 |
| F1 Score  |  0.84 |
| Accuracy  |  0.82 |

Наиболее значимыми факторами риска оттока стали:

* Frequency;
* Monetary Growth;
* Monetary Delta;
* Frequency Growth;
* Avg Days Between Purchases.

Результаты подтверждают, что ухудшение покупательской динамики является одним из наиболее ранних индикаторов потенциального ухода клиента.

#

### Revenue Risk & Retention Prioritization

Анализ показал, что высокий риск оттока сам по себе не является достаточным основанием для retention-инвестиций.

Для поддержки управленческих решений был разработан интегральный Retention Priority Score, объединяющий:

* вероятность оттока;
* ожидаемую будущую ценность клиента;
* вероятность восстановления активности.

Такой подход позволяет перейти от массовых retention-кампаний к приоритизации клиентов по ожидаемому бизнес-эффекту и более эффективно распределять CRM-бюджет.

#

### Business Outcome

Результатом проекта стала комплексная аналитическая система, позволяющая:

* понимать механизмы формирования клиентской ценности;
* выявлять структурные источники потерь выручки;
* прогнозировать churn-risk;
* оценивать revenue-at-risk;
* приоритизировать retention-инвестиции;
* организовать непрерывный мониторинг состояния клиентской базы.

Проект демонстрирует полный цикл Customer & Product Analytics: от описательного анализа и когортной аналитики до предиктивного моделирования и формирования бизнес-ориентированных рекомендаций для принятия решений.

## 1.4 Business Impact

Проведенный анализ показал, что ключевые риски клиентской базы сосредоточены не в полном оттоке пользователей, а в постепенном ухудшении покупательской активности на ранних стадиях жизненного цикла.

Когортный анализ выявил резкое снижение удержания сразу после первой покупки: средний User Retention сокращается примерно до 26% уже ко второму месяцу жизни когорты. При этом после второго месяца скорость оттока существенно замедляется, формируя устойчивое retention plateau. Данный результат указывает на критическую важность первых месяцев взаимодействия с клиентом для долгосрочного удержания.

Анализ миграции клиентов между lifecycle-сегментами показал, что основная часть потерь формируется за счет накопления пользователей в сегменте At Risk. Наиболее значимый источник отрицательной динамики выручки связан с переходом клиентов из At Risk в Churn, тогда как крупнейшим драйвером роста выступает расширение потребления внутри уже удержанной клиентской базы.

Предиктивное моделирование подтвердило возможность раннего выявления клиентов с повышенным риском ухода. Построенная churn-модель продемонстрировала высокий уровень качества (ROC-AUC = 0.89, Recall = 0.95), что позволяет использовать прогнозные оценки для приоритизации retention-активностей и оценки потенциального объема revenue-at-risk.

Дополнительный анализ показал, что не все клиенты с высоким риском оттока обладают одинаковой коммерческой значимостью. Интеграция churn-risk, ожидаемой будущей выручки и вероятности восстановления активности позволила сформировать систему приоритизации retention-инвестиций и выделить клиентов с максимальным ожидаемым эффектом от удержания.

В совокупности результаты исследования формируют основу для перехода от реактивного управления оттоком к системной data-driven retention-стратегии, ориентированной на сохранение наиболее ценных и наиболее «спасаемых» клиентов.


## 1.5 Strategic Priorities

На основании проведенного анализа можно выделить пять ключевых направлений развития retention-стратегии.

### Priority 1. Reduce Early Lifecycle Churn

Наиболее значительная потеря клиентов происходит в первые месяцы после первой покупки.

Приоритетом должно стать повышение раннего удержания через:

* welcome-программы;
* персонализированные onboarding-коммуникации;
* стимулирование второй и третьей покупки;
* сокращение интервалов между первыми транзакциями.

Даже небольшое улучшение раннего retention способно оказать существенное влияние на долгосрочную клиентскую базу и будущую выручку.

#

### Priority 2. Prevent Transition to At Risk Segment

Анализ жизненного цикла показал, что сегмент At Risk выступает основным источником последующего churn.

Для данной группы необходимо внедрять систему раннего предупреждения, основанную на мониторинге:

* снижения частоты покупок;
* ухудшения Monetary Dynamics;
* увеличения интервалов между покупками;
* роста прогнозируемого churn-risk.

Основная задача заключается в предотвращении накопления клиентов в зоне повышенного риска.

#

### Priority 3. Protect High-Value Customer Segments

Несмотря на относительно небольшую численность, сегмент Loyal формирует более половины совокупной выручки клиентской базы.

Для данных клиентов рекомендуется развитие отдельных программ удержания:

* персонализированные предложения;
* программы лояльности;
* VIP-коммуникации;
* механики увеличения частоты покупок.

Сохранение даже небольшой доли высокоценных клиентов способно обеспечить непропорционально высокий экономический эффект.

#

### Priority 4. Prioritize Retention Investments Using Predictive Scoring

Результаты исследования показывают, что высокий риск оттока сам по себе не является достаточным основанием для запуска retention-интервенций.

Приоритизация должна учитывать одновременно:

* вероятность оттока;
* ожидаемую будущую выручку;
* вероятность успешной реактивации клиента.

Использование Retention Priority Score позволяет концентрировать ресурсы на клиентах с максимальным ожидаемым возвратом инвестиций в удержание.

#

### Priority 5. Build Continuous Customer Health Monitoring

Для обеспечения устойчивого контроля клиентской базы рекомендуется внедрение регулярного мониторинга ключевых lifecycle-метрик:

* Retention Rate;
* Churn Rate;
* Reactivation Rate;
* Revenue Retention;
* Revenue-at-Risk;
* Lifecycle Stability Index.

Такой подход позволит своевременно выявлять ухудшение состояния клиентской базы и оценивать эффективность принимаемых retention-решений.

# 2. Business Background

Данный проект основан на данных региональной аптечной сети, объединяющей восемь аптек, расположенных в различных районах города. Компания работает на высококонкурентном рынке розничной фармацевтики, где значительная часть выручки формируется за счет повторных покупок постоянных клиентов.

Несмотря на стабильный поток новых покупателей, руководство сети столкнулось с постепенным снижением повторной активности части клиентской базы. Отсутствие системного понимания жизненного цикла клиентов, факторов оттока и потенциальной ценности отдельных сегментов затрудняло принятие решений о распределении маркетингового бюджета и приоритизации retention-инициатив.

В ответ на данный бизнес-запрос был разработан комплексный аналитический framework, объединяющий анализ жизненного цикла клиентов, когортную аналитику, прогнозирование оттока и систему приоритизации retention-инвестиций для поддержки data-driven управленческих решений.

**Примечание.** Проект выполнен в образовательных целях на обезличенных транзакционных данных. Описание компании и бизнес-контекст являются реалистичным сценарием, созданным для демонстрации применения методов продуктовой и клиентской аналитики.

# 3. Project Architecture

## 3.1 Architecture Overview

### Почему проект построен как многоуровневая аналитическая система

Большинство retention-анализов ограничиваются изучением отдельных метрик: churn rate, retention rate, RFM-сегментации или прогнозированием оттока. Такой подход позволяет выявлять локальные закономерности, однако не формирует целостного понимания жизненного цикла клиента и механизмов изменения выручки.

Для принятия управленческих решений бизнесу необходимо одновременно понимать:

* как изменяется структура клиентской базы;
* какие механизмы формируют удержание и отток клиентов;
* каким образом меняется ценность клиентов во времени;
* какие пользователи представляют наибольший риск для бизнеса;
* где необходимо концентрировать retention-инвестиции;
* какие показатели должны использоваться для постоянного мониторинга клиентского здоровья.

Для решения этой задачи проект построен в формате многоуровневой аналитической архитектуры (Multi-Layer Analytics Framework), где каждый аналитический слой отвечает на отдельный класс бизнес-вопросов и последовательно развивает результаты предыдущего этапа.

Логика исследования построена по принципу постепенного перехода:

							описание поведения клиентов
										↓
							объяснение механизмов удержания
										↓
							прогнозирование будущего поведения 
										↓
							построение системы принятия решений.

# 3.2 Multi-Layer Analytics Framework

### Layer 1. Lifecycle Transition Analytics

**Ключевой вопрос:**

> Как изменяется клиентская база во времени?

Первый слой исследования посвящен анализу динамики жизненного цикла клиентов.

# 4. Business Objectives

## 4.1 Business Context

Аптечная сеть функционирует в условиях высокой конкуренции и снижения покупательской активности, что делает удержание существующих клиентов одним из ключевых факторов устойчивого роста бизнеса. При этом клиентская база неоднородна: пользователи существенно различаются по частоте покупок, уровню расходов, склонности к повторному потреблению и вероятности оттока.

Традиционный анализ продаж позволяет оценивать текущие финансовые результаты, однако не дает ответа на более важные стратегические вопросы: каким образом меняется клиентская база во времени, какие сегменты формируют рост или потерю выручки, насколько устойчиво удержание пользователей и какие клиенты требуют приоритетного внимания со стороны CRM-команд.

Для эффективного управления retention-стратегией компании необходимо перейти от статического анализа клиентских сегментов к комплексному lifecycle-подходу, который позволяет одновременно анализировать динамику поведения клиентов, исследовать механизмы удержания, прогнозировать риск оттока и оценивать потенциальный бизнес-эффект от retention-интервенций.

Особую значимость представляет понимание взаимосвязи между жизненным циклом клиента, динамикой удержания, будущей ценностью пользователей и потенциальными потерями выручки. Именно эта информация позволяет принимать решения о распределении маркетингового бюджета, приоритизации CRM-активностей и управлении долгосрочной устойчивостью клиентской базы.

Для решения данных задач в рамках проекта была разработана многоуровневая аналитическая система, объединяющая lifecycle analytics, cohort retention analysis, predictive customer analytics и customer health monitoring в единый framework поддержки бизнес-решений.

# 5. Dataset Overview

## 5.1 Dataset Description

Исследование выполнено на основе транзакционных данных аптечной сети, содержащих историю покупок клиентов за период с **12 июля 2021 года по 9 июня 2022 года**.

В отличие от традиционного анализа продаж, исследование ориентировано не на отдельные транзакции, а на динамику изменения клиентской базы во времени. Для этого на основе исходных данных формируется система последовательных lifecycle snapshots, которая используется в качестве единой аналитической основы для всех последующих этапов проекта.

## 7.1 Business Objective

Статический анализ клиентской базы позволяет понять текущее распределение пользователей по сегментам, однако практически не отвечает на вопрос о том, как именно формируется рост или снижение бизнеса во времени. Для аптечной сети критически важно понимать не только текущее состояние клиентского портфеля, но и механизмы его изменения:  
-	каким образом клиенты переходят между стадиями жизненного цикла;   
-	какие сегменты демонстрируют наиболее устойчивое поведение;   
-	где возникают основные потери клиентов;  
-	какие изменения пользовательского поведения приводят к росту или снижению выручки.  

Особую важность представляет анализ переходов между lifecycle-состояниями, поскольку именно они позволяют выявлять ранние признаки деградации клиентской базы задолго до фактического оттока.
#
### Цель анализа

Цель данного раздела — построить систему Lifecycle Transition Analytics, позволяющую количественно оценить динамику клиентской базы и связанные с ней денежные потоки. В рамках анализа необходимо:  
-	исследовать миграцию клиентов между lifecycle-сегментами;   
-	оценить устойчивость различных состояний жизненного цикла;   
-	выявить основные сценарии деградации и удержания клиентов;   
-	определить ключевые источники роста и потери выручки;   
-	проанализировать вклад *retention*, *reactivation*, *expansion*, *contraction* и *churn* в изменение финансовых результатов бизнеса.

Основная задача данного этапа — определить, насколько устойчивы различные lifecycle-сегменты и какие сценарии миграции формируют долгосрочную структуру клиентской базы.

### Ключевые результаты

Анализ агрегированной матрицы переходов выявил существенные различия в устойчивости различных lifecycle-сегментов.
Наиболее стабильным состоянием среди активной клиентской базы оказался сегмент Loyal. Большинство клиентов, достигших данной стадии, сохраняют высокий уровень вовлеченности и продолжают совершать покупки в последующих периодах.
Сегмент Churn также характеризуется высокой стабильностью, однако уже в негативном смысле: подавляющее большинство клиентов, перешедших в состояние оттока, не возвращаются к активному потреблению.

Сегмент Active выступает ключевой точкой формирования будущей динамики клиентской базы. Несмотря на высокую долю клиентов, сохраняющих статус Active, вероятность перехода в состояние At Risk существенно превышает вероятность перехода в сегмент Loyal.
Сегмент At Risk выполняет роль промежуточного состояния между активным потреблением и окончательным оттоком. Для клиентов данной группы вероятность дальнейшей деградации оказывается выше вероятности возврата к стабильной активности.

Для новых клиентов наблюдается высокий уровень первичной активации: большинство пользователей успешно переходят в сегмент Active после первой покупки. Однако лишь небольшая часть сразу формирует устойчивое лояльное поведение.
#
### Бизнес-интерпретация

Полученные результаты показывают, что отток клиентов является не разовым событием, а постепенным процессом поведенческой деградации.

    Наиболее типичный сценарий развития негативной динамики:
    Active  →  At Risk  →  Churn

Таким образом, признаки будущего оттока начинают проявляться задолго до того момента, когда клиент фактически прекращает взаимодействие с компанией.
Одновременно анализ показывает, что сегмент Loyal является основным источником долгосрочной устойчивости клиентской базы. После формирования регулярного покупательского поведения вероятность дальнейшего удержания клиента существенно возрастает.
Это позволяет рассматривать lifecycle transitions как ранние индикаторы будущего состояния клиентского портфеля.
#
### Стратегические выводы

С точки зрения управления удержанием особое внимание следует уделять трем переходам:

    • Active → At Risk  
    Основной источник формирования будущего оттока.

    • At Risk → Churn  
    Критический этап окончательной потери клиента.

    • Active → Loyal  
    Наиболее важный путь формирования долгосрочной клиентской ценности.

Полученные результаты указывают на необходимость смещения retention-активностей в более ранние стадии жизненного цикла, когда изменения поведения еще являются обратимыми.
#
### Вывод по разделу Lifecycle Transition Framework

Переходы между lifecycle-состояниями формируют устойчивую и предсказуемую структуру поведения клиентской базы.
Большая часть клиентов перемещается между ограниченным набором типовых состояний, причем основные потери клиентов происходят через постепенную деградацию в сегменте At Risk, а основная клиентская ценность концентрируется в сегменте Loyal.
Следовательно, мониторинг lifecycle transitions может использоваться как инструмент раннего выявления рисков и как основа для построения проактивной retention-стратегии.

## 7.6 Revenue Driver Framework

#
### Driver Classification Model

Все изменения выручки были распределены по пяти категориям:    
- *Acquisition* - Выручка от новых клиентов.

- *Expansion* - Рост выручки внутри существующей клиентской базы.  
- *Contraction* - Снижение выручки среди удержанных клиентов.  
- *Churn* - - Потери выручки вследствие прекращения активности клиентов.

Рост бизнеса формируется через acquisition, reactivation и expansion, тогда как основные потери возникают вследствие contraction и churn.
Построенный Revenue Driver Framework обеспечивает прозрачную связь между изменением поведения клиентов и финансовыми результатами компании, создавая основу для принятия retention-ориентированных управленческих решений.

## 7.7 Retained Customer Economics
### Цель анализа

Предыдущие этапы исследования показали, каким образом клиенты перемещаются между lifecycle-состояниями и как эти переходы влияют на общую динамику выручки.
Однако изменение финансовых результатов бизнеса определяется не только удержанием клиентов как таковым.
Даже среди клиентов, которые остаются активными, могут происходить существенные изменения покупательского поведения:
-	часть пользователей увеличивает объем потребления; 
-	часть пользователей снижает интенсивность покупок; 
-	часть клиентов сохраняет относительно стабильный уровень потребления. 

Поэтому следующим этапом анализа проводится исследование экономики удержанной клиентской базы.
Основная задача данного этапа — определить, формируют ли удержанные клиенты дополнительный рост выручки или, наоборот, становятся источником постепенной деградации бизнеса.
#
### Методология

Для клиентов, сохранивших активность между последовательными периодами наблюдения, были рассчитаны изменения выручки относительно предыдущего периода.
Все удержанные клиенты были классифицированы по трем типам поведения:  
- *Expansion* - Рост выручки относительно предыдущего периода.  
- *Stable* - Сохранение сопоставимого уровня потребления.  
- *Contraction* - Снижение выручки относительно предыдущего периода.

Такой подход позволяет оценить качество удержания клиентской базы, а не только факт ее сохранения.

#
### Ключевые результаты

Анализ показал, что удержание клиентов не является однородным процессом.
Даже внутри сохраненной клиентской базы наблюдаются существенные различия в динамике потребления.
Часть удержанных пользователей демонстрирует устойчивый рост покупательской активности и формирует положительный вклад в изменение выручки компании.
Одновременно значительная доля клиентов остается активной формально, однако постепенно сокращает объем потребления, генерируя скрытые потери выручки.
Таким образом, удержание клиента само по себе не гарантирует сохранение его экономической ценности.
#
### Бизнес-интерпретация

Полученные результаты показывают, что традиционные показатели retention отражают лишь количественную сторону удержания.
Фактическая экономическая эффективность retention зависит от качества удержанной клиентской базы.
Компания может демонстрировать высокие показатели удержания пользователей и одновременно сталкиваться с ухудшением финансовых результатов вследствие постепенного снижения активности существующих клиентов.
Поэтому анализ retained customer economics позволяет выявлять риски, которые остаются незаметными при использовании только классических retention-метрик.
#
### Стратегические выводы

Наибольшую ценность для бизнеса представляет не просто удержание клиентов, а увеличение их долгосрочной потребительской активности. С точки зрения управления клиентской базой особое внимание следует уделять:
-	стимулированию expansion среди активных клиентов; 
-	раннему выявлению contraction-поведения; 
-	разработке программ развития клиентской ценности; 
-	контролю скрытых потерь выручки внутри удержанной базы. 
#
### Вывод по разделу Retained Customer Economics
Анализ retained customer economics показал, что значительная часть финансовой динамики бизнеса формируется внутри уже удержанной клиентской базы.
Следовательно, управление ростом бизнеса требует не только предотвращения churn, но и систематического развития покупательской активности существующих клиентов.

## 7.8 Key Findings & Business Implications
### Executive Summary

Проведенный анализ показал, что динамика клиентской базы формируется через ограниченное количество устойчивых поведенческих сценариев.
Lifecycle-переходы демонстрируют высокую степень предсказуемости, что позволяет использовать их в качестве ранних индикаторов будущих изменений retention и revenue dynamics.
#
### Key Insights
### 1. Отток клиентов — это постепенный процесс.
Основная часть клиентских потерь формируется не одномоментно, а через последовательную деградацию поведения.
Наиболее характерный сценарий выглядит следующим образом:

    Active 
      ↓
    At Risk
      ↓
    Churn
    
Это означает, что признаки будущего оттока возникают значительно раньше фактической потери клиента.
#
### 2. Лояльные клиенты — основа стабильности бизнеса.  
Сегмент Loyal демонстрирует максимальную устойчивость среди активных lifecycle-состояний.
Клиенты, достигшие данной стадии, обладают значительно более высокой вероятностью дальнейшего удержания и формируют основу долгосрочной клиентской ценности.
#
### 3. Снижение выручки происходит до оттока клиентов. 
Финансовые потери начинают формироваться задолго до окончательного ухода клиента.
Переходы в сегмент At Risk сопровождаются заметным ухудшением потребительского поведения и снижением объема покупок.
В результате существенная часть revenue loss возникает еще до фактического перехода клиентов в состояние Churn.
#
### 4. Существующие клиенты в значительной степени определяют динамику доходов.
Изменение выручки определяется не только привлечением новых пользователей.
Существенную роль играют процессы expansion и contraction внутри существующей клиентской базы.
Это означает, что развитие отношений с текущими клиентами оказывает не меньшее влияние на финансовые результаты, чем acquisition.
#
### 5. Lifecycle-переходы могут использоваться в качестве опережающих индикаторов. 
Высокая устойчивость transition patterns позволяет использовать lifecycle-переходы в качестве инструмента раннего прогнозирования изменений клиентской базы.
Мониторинг переходов между lifecycle-состояниями способен выявлять ухудшение customer health существенно раньше традиционных показателей churn.
#
### Business Implications

Полученные результаты формируют несколько важных управленческих выводов.

**Retention Strategy**  
Фокус удержания должен смещаться в сторону раннего выявления клиентов, демонстрирующих признаки перехода из Active в At Risk.

**Revenue Protection**  
Для минимизации потерь выручки необходимо контролировать не только churn, но и процессы постепенной деградации покупательской активности внутри удержанной базы.

**Customer Development**  
Рост бизнеса зависит не только от acquisition, но и от способности компании развивать существующих клиентов и стимулировать expansion-поведение.

**CRM Prioritization**  
Lifecycle-состояния могут использоваться как основа для приоритизации CRM-активностей и персонализированных retention-механик.

## Итог раздела Lifecycle Transition Analytics

В рамках данного этапа была построена система Lifecycle Transition Analytics, объединяющая анализ клиентской миграции, денежных потоков и драйверов изменения выручки.
Проведенное исследование показало, что изменение финансовых результатов бизнеса является прямым следствием изменений клиентского поведения и может быть объяснено через ограниченный набор lifecycle-переходов и revenue drivers.
Полученные результаты формируют основу для следующего этапа исследования — анализа долгосрочного удержания и формирования customer value на уровне когорт пользователей.

# 8. Layer 2. Cohort Retention & Customer Value Dynamics

## 8.1 Business Objective

### Бизнес-контекст

Для большинства компаний привлечение новых клиентов является дорогостоящим процессом, поэтому долгосрочная устойчивость бизнеса во многом определяется способностью удерживать существующую клиентскую базу и увеличивать ценность клиентов после первой покупки. Однако традиционные показатели продаж позволяют оценить лишь текущие финансовые результаты и практически не отвечают на вопрос о том, как развивается поведение клиентов после привлечения.  

Для аптечной сети особенно важно понимать:  
-	насколько быстро пользователи прекращают совершать повторные покупки;   
-	существуют ли устойчивые модели долгосрочного потребления;   
-	как меняется покупательская активность по мере взросления клиентов;   
-	каким образом формируется долгосрочная клиентская ценность;   
-	какие когорты демонстрируют наиболее устойчивое поведение.

Для решения этих задач необходим переход от анализа отдельных транзакций к когортному подходу, позволяющему отслеживать жизненный цикл клиентов с момента первой покупки и анализировать их поведение во времени.  
#  
### Цель анализа

Цель данного раздела — исследовать долгосрочную динамику удержания клиентов и формирования customer value на уровне когорт пользователей. В рамках анализа необходимо:    
-	оценить скорость снижения клиентской активности после первой покупки;  
-	определить критические этапы жизненного цикла, на которых происходит максимальный отток;   
-	исследовать устойчивость удержания различных когорт;   
-	проанализировать изменение выручки удержанных клиентов во времени;   
-	оценить динамику customer value после привлечения;   
-	определить степень различий между когортами и устойчивость выявленных закономерностей.   
#
### Ключевые бизнес-вопросы

Анализ направлен на получение ответов на следующие вопросы:  
-	Как быстро снижается активность клиентов после первой покупки?   
-	На каких этапах жизненного цикла происходит основной отток?   
-	Формируется ли устойчивое повторное покупательское поведение?   
-	Насколько отличаются показатели удержания между различными когортами?   
-	Как изменяется выручка удержанных клиентов во времени?    
-	Растет или снижается ценность клиента по мере увеличения его жизненного цикла?  
-	Насколько стабильны наблюдаемые закономерности между различными поколениями клиентов?   
#
### Бизнес-ценность  

В отличие от классического retention-анализа данный подход позволяет исследовать не только факт удержания клиентов, но и процесс формирования долгосрочной клиентской ценности. Полученные результаты создают основу для:    
-	оценки качества привлечения клиентов;     
-	понимания экономики жизненного цикла пользователя;    
-	разработки retention-стратегий;   
-	оптимизации CRM-коммуникаций;   
-	повышения эффективности маркетинговых инвестиций;   
-	последующего построения прогнозных моделей customer lifetime value и churn.   

## 8.2 Analytical Approach

### Общая концепция анализа

Для исследования долгосрочной динамики клиентского поведения был реализован cohort-based analytical framework, позволяющий отслеживать эволюцию клиентской базы с момента первой покупки. В отличие от lifecycle-анализа предыдущего раздела, где основным объектом исследования являлись переходы между состояниями жизненного цикла, данный раздел фокусируется на поведении клиентских когорт и изменении их ценности во времени. Каждая когорта представляет собой группу пользователей, совершивших первую покупку в одном и том же календарном периоде. Такой подход позволяет сравнивать клиентов на одинаковых стадиях жизненного цикла независимо от даты их привлечения.  
#
### Формирование когорт

Для каждого клиента были определены:  
-	дата первой покупки;   
-	месяц привлечения;   
-	номер периода относительно первой покупки (cohort age).

На основе этих признаков пользователи были объединены в когорты по месяцу первого взаимодействия с компанией. Для каждого последующего месяца жизненного цикла рассчитывались показатели активности и монетизации пользователей.  
#
### Аналитический Framework

Исследование построено вокруг пяти взаимосвязанных аналитических блоков.

### 1. Cohort Retention Analysis  

На первом этапе анализируется динамика удержания клиентов. Для каждой когорты рассчитывается доля пользователей, сохранивших активность через определенное количество месяцев после первой покупки. Данный блок позволяет определить:  
-	скорость первоначального оттока;   
-	устойчивость удержания;   
-	точки стабилизации поведения клиентов;  
-	различия между когортами.   
#
### 2. Retention Decay Analysis  

Следующим этапом исследуется скорость ухудшения удержания во времени. Помимо абсолютных значений retention дополнительно рассчитывается динамика изменения показателей между соседними периодами жизненного цикла. Это позволяет определить:    
-	когда происходит наиболее интенсивная потеря клиентов;   
-	насколько быстро замедляется отток;   
-	существует ли стабильное ядро удержанных пользователей.   
#
### 3. Revenue Retention Dynamics   

После анализа удержания пользователей исследуется динамика выручки. Для каждой когорты рассчитывается уровень revenue retention относительно периода первой покупки. Это позволяет определить:  
-	как изменяется финансовая ценность удержанной базы;   
-	совпадают ли динамика пользователей и динамика выручки;   
-	сохраняется ли экономическая ценность клиентов по мере старения когорт.   
#
### 4. Customer Value Evolution 

Финальным этапом является исследование изменения customer value среди удержанных клиентов. Для этого рассчитывается показатель *Revenue per Retained User (RPRU)*, отражающий среднюю выручку на одного удержанного пользователя. Данный показатель позволяет оценить:   
-	качество удержания клиентской базы;   
-	изменение покупательской активности;   
-	долгосрочную динамику клиентской ценности;   
-	потенциал формирования устойчивой пользовательской экономики.   
#
### 5. Cohort Variability Analysis

После анализа удержания и монетизации необходимо оценить устойчивость выявленных закономерностей между различными когортами пользователей.  
Даже если средние показатели retention и revenue демонстрируют определенную тенденцию, важно понять:    
-	насколько различаются отдельные когорты;   
-	являются ли наблюдаемые закономерности стабильными;   
-	не обусловлены ли результаты поведением нескольких крупных когорт.  

Для этого дополнительно анализируются:  
-	cohort dispersion;   
-	confidence bands;   
-	behavioral heterogeneity.

Данный этап позволяет отделить структурные закономерности клиентского поведения от случайных колебаний отдельных когорт.
#
### Проверка устойчивости результатов

Для минимизации влияния статистического шума и эффекта незрелых когорт дополнительно применялись процедуры контроля надежности результатов.
В частности использовались:
-	mature cohort filtering; 
-	cohort variability analysis; 
-	стандартное отклонение; 
-	доверительные интервалы.

Данные подходы позволяют убедиться, что выявленные закономерности отражают устойчивые особенности клиентского поведения, а не случайные колебания отдельных когорт.
#
### Логика исследования

Исследование построено как последовательный анализ жизненного цикла клиента — от удержания пользователей после первой покупки до формирования долгосрочной клиентской ценности.

      Cohort Formation
              ↓
      Retention Analysis
              ↓
      Retention Decay Analysis
              ↓
      Revenue Retention Analysis
              ↓
      Customer Value Evolution
              ↓
      Cohort Variability Assessment
              ↓
      Long-Term Customer Value Framework

Такой подход позволяет последовательно перейти от оценки удержания клиентов к пониманию механизмов формирования долгосрочной ценности клиентской базы и устойчивости выявленных закономерностей между когортами.

## 8.3 Cohort Retention Analysis

### Business Question

Как изменяется удержание клиентов после первой покупки и существует ли устойчивая база пользователей, формирующая долгосрочную ценность бизнеса?

### Methodology 

Для каждого клиента был определен месяц первой покупки (cohort month), после чего пользователи были объединены в месячные когорты. Для каждой когорты рассчитывалась доля клиентов, совершивших хотя бы одну покупку в последующие месяцы жизненного цикла. Анализ выполнялся на двух уровнях:  
-	по отдельным когортам;   
-	в агрегированном виде по всем когортам.  
 
Это позволило оценить как общую retention-модель бизнеса, так и различия между поколениями клиентов.

Анализ retention-кривых выявил выраженный ранний отток клиентов практически во всех когортах. Средний User Retention снижается со 100% до примерно 19–34% уже после первого месяца жизненного цикла. Основная потеря клиентской базы происходит практически сразу после первой покупки. При этом пользователи, сохранившие активность после первых месяцев жизненного цикла, демонстрируют существенно более высокий уровень долгосрочного удержания. Фактически именно прохождение раннего post-acquisition периода определяет вероятность формирования долгосрочной клиентской ценности.

## 8.4 Retention Decay Analysis
### Business Question

В какие моменты жизненного цикла компания теряет наибольшую долю клиентов и насколько быстро замедляется процесс оттока?
#
### Methodology

Для оценки скорости ухудшения удержания были рассчитаны две взаимодополняющие метрики:  

- *Absolute Retention Decay*    
Абсолютное изменение retention между соседними месяцами жизненного цикла.  

- *Relative Retention Decay*  
Относительное изменение retention относительно предыдущего периода.  

Использование обеих метрик позволяет оценить не только величину потерь клиентов, но и скорость деградации retention-модели.
#

Наиболее существенное снижение retention наблюдается между первым и вторым месяцем жизни клиента.

Практически весь будущий объем клиентской базы определяется в течение первых недель после первой покупки.

### Core Finding

Retention-модель бизнеса имеет ярко выраженную двухфазную структуру:

    Acquisition  →  Aggressive Early Churn  →  Retention Stabilization  →  Long-Term Customer Base

Большая часть будущего LTV определяется не качеством удержания зрелых клиентов, а способностью провести пользователя через фазу раннего churn.
#
### Business Implications

Результаты показывают, что наиболее существенная потеря клиентов происходит в первые месяцы после первой покупки, когда скорость снижения User Retention достигает максимальных значений. По мере развития жизненного цикла величина ежемесячных потерь постепенно уменьшается, что свидетельствует о формировании более устойчивой клиентской базы. 

После прохождения точки стабилизации дальнейшее снижение retention становится значительно менее выраженным, а поведение оставшихся клиентов приобретает более предсказуемый характер. При этом Revenue Retention стабилизируется на более высоком уровне, чем User Retention, что указывает на сохранение экономически наиболее ценной части клиентской базы. Данный результат подтверждает, что долгосрочная выручка компании формируется преимущественно за счет относительно небольшого ядра наиболее лояльных и регулярно покупающих клиентов. Выявленная точка стабилизации выступает важным элементом customer lifecycle model и служит естественным связующим звеном к дальнейшему анализу монетизации удержанных клиентов, динамики RPRU и формирования долгосрочного Customer Lifetime Value.
#
### Strategic Implications

Наибольший потенциальный эффект для бизнеса способны обеспечить инициативы, направленные на:  
-	увеличение вероятности второй покупки;   
-	ускорение формирования покупательской привычки;   
-	персонализированное сопровождение новых клиентов;   
-	CRM-механики в первые 30–60 дней после привлечения.  

Даже небольшое снижение churn на данном этапе способно оказать непропорционально сильное влияние на:  
-	размер активной клиентской базы;   
-	cumulative revenue;   
-	customer lifetime value;   
-	окупаемость привлечения клиентов.

Сохраняется ли выручка клиентских когорт с той же скоростью, что и пользовательская база, или удержанные клиенты со временем концентрируют все большую долю дохода компании?

-	в сравнении с User Retention.

Следовательно, основной резерв роста заключается не в дополнительной оптимизации монетизации существующих loyal customers, а в увеличении числа клиентов, которые достигают стадии повторного потребления.

## 8.6 Customer Value Evolution (RPRU)
### Business Question

#
### Methodology

Для оценки качества удержанной аудитории рассчитывался показатель *Revenue per Retained User (RPRU)*. Метрика отражает среднюю выручку на одного активного пользователя внутри каждой когорты на каждом этапе жизненного цикла. Использование RPRU позволяет отделить изменения количества пользователей от изменений их экономической ценности.

## 8.7 Retention vs Monetization Dynamics

### Business Question

Что оказывает большее влияние на долгосрочную ценность клиента и рост выручки компании:
-	удержание пользователей; 
-	рост монетизации удержанной аудитории? 
#
### Methodology

Для ответа на данный вопрос были объединены результаты предыдущих этапов анализа:
-	User Retention;   
-	Revenue Retention;   
-	Revenue per Retained User (RPRU).  
 
Сопоставление этих метрик позволяет разделить два независимых механизма формирования выручки:

- Retention Effect  
Изменение количества активных пользователей.

- Monetization Effect  
Изменение средней выручки на одного удержанного клиента.

Такой подход позволяет определить, какой из факторов оказывает наибольшее влияние на долгосрочную экономику клиентской базы.
#
### Retention vs Revenue Dynamics

### Findings

Сравнение user retention и revenue retention [Figure 25](#fig25) выявило устойчивое расхождение между динамикой клиентской базы и динамикой выручки. После первого месяца жизненного цикла пользовательская база сокращается примерно до 26% первоначального размера, тогда как Revenue Retention остается существенно выше и сохраняет значительную часть исходной выручки. Это означает, что потери пользователей не приводят к пропорциональному снижению финансовой ценности клиентской базы. По мере развития жизненного цикла происходит концентрация выручки среди меньшего количества клиентов.
#
### Retention vs Customer Value

#### *Figure 29. Ценность удержанных клиентов растет, но этого недостаточно, чтобы компенсировать потери аудитории.*
*Средний User Retention и индекс роста Revenue per Retained User (RPRU) по месяцам жизни когорты.*

![Average User Retention vs Average RPRU Growth](visualizations/retention_vs_monetization.png)

### Findings

Анализ показывает, что по мере сокращения клиентской базы средняя ценность удержанных пользователей систематически растет.
После прохождения периода интенсивного churn Revenue per Retained User превышает уровень первого месяца и продолжает увеличиваться на протяжении жизненного цикла. Для зрелых когорт рост RPRU достигает 20–40%, а в отдельных случаях превышает 40%. Таким образом, система одновременно демонстрирует:   
-	сокращение количества клиентов;   
-	рост ценности оставшейся аудитории.  

### Interpretation

Наблюдаемая динамика свидетельствует о механизме естественной селекции клиентской базы. Низкововлеченные пользователи покидают продукт на ранних стадиях жизненного цикла, тогда как наиболее активные покупатели продолжают взаимодействовать с компанией и увеличивают свой вклад в выручку. В результате продукт постепенно концентрирует все большую долю дохода внутри сегмента лояльных клиентов.
#
### Growth Constraint Analysis
### Findings

Несмотря на устойчивый рост RPRU, увеличение ценности удержанных клиентов не компенсирует масштаб первоначального churn. Средний рост монетизации на одного retained user составляет порядка 15–40%. Одновременно клиентская база сокращается более чем на 70% уже в первые месяцы жизненного цикла.
Это означает, что потери клиентов значительно превосходят выигрыш от роста монетизации удержанной аудитории.

### Core Insight

Главным ограничителем роста бизнеса является не монетизация удержанных клиентов. Главным ограничителем роста является способность компании переводить новых покупателей в сегмент повторного потребления.
#
### Business Implications

Полученные результаты показывают, что продукт уже обладает эффективной моделью монетизации удержанных пользователей. Основной резерв роста находится в расширении сегмента клиентов, достигающих стадии высокой customer value. Следовательно, наиболее перспективными направлениями развития являются:    
-	улучшение раннего retention;   
-	увеличение вероятности второй покупки;   
-	ускорение формирования покупательской привычки;   
-	развитие программ повторного вовлечения новых клиентов.   

## 8.8 Cohort LTV Dynamics

Как совокупное влияние удержания и монетизации формирует долгосрочную ценность клиента и каким образом развивается LTV различных когорт во времени?
#
### Methodology
Для каждой acquisition-когорты был рассчитан накопленный Customer Lifetime Value (LTV). Метрика отражает совокупную выручку, генерируемую клиентом с момента первой покупки на протяжении жизненного цикла.  
Дополнительно анализировались:

### Core Insight

Даже при относительно низком уровне удержания продукт способен генерировать значительную долгосрочную ценность благодаря устойчивой монетизации retained-аудитории. Непрерывный рост кумулятивного LTV показывает, что пользователи, пережившие ранние этапы жизненного цикла, продолжают приносить компании дополнительную выручку в течение длительного времени. Однако итоговый LTV определяется прежде всего тем, сколько клиентов смогут пройти через период наиболее интенсивного churn в первые месяцы использования продукта.
#
### Business Implications

Полученные результаты позволяют сделать стратегически важный вывод. Рост долгосрочной выручки компании зависит не столько от дальнейшего повышения интенсивности трат существующих loyal клиентов, сколько от увеличения числа клиентов, достигающих стадий жизненного цикла с высокой клиентской ценностью. Каждый дополнительный процент retention в первые месяцы жизненного цикла оказывает существенно большее влияние на итоговый LTV, чем аналогичное улучшение удержания на поздних этапах. Дополнительно результаты указывают на высокий потенциальный эффект инвестиций в onboarding, активацию и раннее удержание пользователей, поскольку именно на этих этапах формируется основная часть будущей клиентской ценности.

## 8.9 Cohort Variability Analysis

### Business Question

Насколько выявленные закономерности retention и монетизации являются устойчивыми для всей клиентской базы, а не результатом поведения отдельных когорт?
#
### Почему этот анализ важен

До этого момента мы анализировали средние retention-кривые, revenue retention и динамику customer value. Однако средние значения могут скрывать существенные различия между когортами.  
Для принятия бизнес-решений важно понимать:  
-	являются ли выявленные закономерности структурными;   
-	насколько различаются отдельные acquisition cohorts;   
-	сохраняются ли выводы при анализе разных периодов привлечения клиентов;  
-	не формируется ли средний результат за счет нескольких крупных когорт.  
#
### Analytical Approach

Для оценки устойчивости результатов дополнительно исследуется межкогортная вариативность поведения.
Анализ включает три уровня проверки:

***Cohort Dispersion***  
Оценка разброса retention между когортами на различных стадиях жизненного цикла.  
Позволяет определить:
-	насколько похожи разные когорты; 
-	увеличивается ли неоднородность поведения со временем; 
-	существуют ли когорты с аномально высоким или низким удержанием. 

***Confidence Bands***  
Для средних retention curves рассчитываются доверительные интервалы.  
Это позволяет оценить:  
-	статистическую устойчивость наблюдаемых закономерностей;   
-	степень неопределенности на поздних lifecycle-этапах;   
-	надежность интерпретации retention plateau.   

***Behavioral Heterogeneity***     
Дополнительно анализируется изменение вариативности:  
-	User Retention;     
-	Revenue Retention;   
-	Revenue per Retained User.
  
Цель анализа — определить, становится ли клиентская база более или менее однородной по мере развития жизненного цикла.

### Findings

Анализ межкогортной вариативности показал, что паттерны удержания клиентов остаются исключительно устойчивыми между различными когортами приобретения.

Практически все когорты демонстрируют одинаковую структуру жизненного цикла:  
- резкое снижение активности после первой покупки;  
- быстрое замедление скорости оттока;  
- формирование retention плато на последующих этапах жизненного цикла.  

Наиболее низкий уровень вариативности наблюдается у пользовательского удержания. Стандартное отклонение User Retention на протяжении большей части жизненного цикла остается в диапазоне 4–6 п.п., что свидетельствует о высокой воспроизводимости retention-паттернов между когортами. В отличие от пользовательского удержания, метрики монетизации демонстрируют существенно более высокий разброс. Для Revenue Retention стандартное отклонение достигает 10–14 п.п., тогда как для Revenue per Retained User наблюдается значительный разброс значений; стандартное отклонение в отдельные периоды превышает 0.45–0.50 (в единицах RPRU).

Это означает, что различия между когортами формируются не столько за счет различий в вероятности повторных покупок, сколько за счет различий в интенсивности потребления удержанных клиентов. Дополнительно наблюдается рост неопределенности на поздних стадиях жизненного цикла. Однако оценки после шестого месяца следует интерпретировать с осторожностью, поскольку в расчете участвует менее пяти зрелых когорт, что увеличивает чувствительность результатов к поведению отдельных групп клиентов.

В целом результаты подтверждают, что выявленные ранее закономерности retention являются структурной характеристикой клиентской базы, тогда как долгосрочная экономика клиентов существенно сильнее зависит от качества удержанной аудитории.
#
### Business Interpretation

Полученные результаты подтверждают высокую устойчивость ключевых выводов всего когортного анализа.

Независимо от месяца привлечения клиентов наблюдаются одни и те же фундаментальные закономерности:
- высокий ранний churn после первой покупки;  
- постепенное замедление скорости оттока;  
- формирование стабильного retention plateau;  
- рост экономической ценности удержанных клиентов.  

Особенно важным является тот факт, что вариативность User Retention остается низкой практически на всем горизонте наблюдения. Это позволяет рассматривать ранний churn как системную особенность клиентской базы, а не как результат поведения отдельных когорт. В то же время основные различия между когортами проявляются в метриках монетизации. Revenue Retention и Revenue per Retained User демонстрируют существенно больший разброс, что указывает на различия в составе удержанной аудитории и доле высокоценных клиентов внутри отдельных когорт. 

Иными словами, когорты теряют клиентов примерно одинаково, но удержанные пользователи обладают разной экономической ценностью. При интерпретации поздних lifecycle-периодов необходимо учитывать снижение статистической устойчивости оценок. После шестого месяца количество зрелых когорт становится ограниченным, поэтому наблюдаемая вариативность частично отражает эффект малой выборки.
#
### Strategic Implications

Полученные результаты имеют несколько важных практических следствий для бизнеса.

1. Высокий ранний churn является структурной характеристикой всей клиентской базы.Поскольку данный паттерн воспроизводится практически во всех когортах, приоритетом retention-стратегии должны стать мероприятия, направленные на удержание клиентов в первые месяцы после первой покупки.

2. Рост customer value среди retained users наблюдается во всех зрелых когортах, что подтверждает экономическую целесообразность инвестиций в удержание пользователей с высоким потенциалом долгосрочной ценности.

3. Основным источником различий между когортами становится не удержание как таковое, а монетизация удержанных клиентов. Это означает, что наряду с задачами retention бизнесу необходимо развивать механизмы повышения глубины потребления и расширения клиентской ценности внутри удержанной аудитории.

4. Долгосрочная выручка компании становится все более зависимой от относительно небольшой группы наиболее ценных клиентов. По мере созревания когорт влияние high-value customers на агрегированные показатели возрастает, что делает их удержание одним из ключевых факторов устойчивости бизнеса.

Таким образом, результаты анализа подтверждают, что управление customer value является не менее важным направлением развития, чем управление удержанием клиентов, а максимальный бизнес-эффект достигается при одновременной работе над обеими задачами.

## 8.10 Key Findings & Business Implications
### Executive Summary

Когортный анализ показал, что долгосрочная экономика клиентской базы определяется не столько масштабом привлечения пользователей, сколько способностью компании переводить новых покупателей в сегмент повторных клиентов. Несмотря на значительный ранний churn, удержанные пользователи демонстрируют устойчивый рост customer value и формируют непропорционально большую долю долгосрочной выручки. Таким образом, основным ограничением роста бизнеса является не монетизация существующих клиентов, а эффективность раннего удержания после первой покупки.
#
### Key Insights

### 1. Основной churn происходит сразу после первой покупки  
User Retention снижается примерно до 26% уже в течение первых месяцев жизненного цикла.
Именно этот период формирует основную потерю будущего LTV.
#
### 2. После initial churn формируется стабильное ядро клиентов   
После первых месяцев скорость оттока резко снижается.
Оставшиеся пользователи демонстрируют признаки устойчивого повторного покупательского поведения.
#
### 3. Revenue Retention устойчивее User Retention  
Выручка сокращается медленнее, чем клиентская база.
Это означает, что удержанные пользователи обладают более высокой экономической ценностью.
#
### 4. Customer Value растет по мере развития жизненного цикла  
Revenue per Retained User систематически увеличивается относительно acquisition period.
На поздних этапах жизненного цикла retained customers генерируют значительно больший revenue contribution.
#
### 5. LTV формируется преимущественно в первые месяцы жизненного цикла       
Наиболее значимая часть будущей ценности клиента определяется в период максимального churn.
Каждое улучшение раннего retention оказывает накопительный эффект на долгосрочную выручку.

## Business Implications
Полученные результаты свидетельствуют о наличии двухфазной структуры жизненного цикла клиентов. Первая фаза характеризуется интенсивным оттоком новых покупателей и определяет размер будущей клиентской базы. Вторая фаза связана с развитием удержанных пользователей, которые постепенно увеличивают свою ценность для бизнеса. В результате долгосрочный рост выручки определяется не столько способностью компании привлекать новых клиентов, сколько эффективностью перевода новых покупателей в сегмент регулярных пользователей.

## Strategic Recommendations

### Приоритет 1. Оптимизация Early Retention     
Максимальный бизнес-эффект ожидается от снижения churn в первые 30–60 дней после первой покупки.

### Приоритет 2. Ускорение повторных покупок   
Необходимо развивать механики, ускоряющие вторую и третью покупку клиента.

### Приоритет 3. Развитие ценных клиентов   
Следует выявить факторы, связанные с ростом RPRU и долгосрочного LTV.

### Приоритет 4. Мониторинг качества когорт  
Рекомендуется регулярно отслеживать качество новых когорт через retention, revenue retention и cohort LTV.

Следовательно, ключевой резерв роста находится не в увеличении монетизации существующей лояльной аудитории, а в повышении вероятности перехода новых клиентов в сегмент долгосрочно удерживаемых пользователей. # 9. Layer 3. Predictive Customer Analytics
## 9.1 Business Objective
### Цель раздела

Первые два раздела проекта позволили понять, каким образом формируется жизненный цикл клиента, какие сегменты наиболее подвержены деградации и где возникают основные потери выручки.

Однако описательный анализ отвечает преимущественно на вопрос:
> Что произошло?

Для построения полноценной retention-стратегии необходимо перейти к следующему уровню аналитики и ответить на вопрос:

Цель данного раздела заключается в построении системы прогнозирования клиентского поведения, позволяющей заранее выявлять пользователей с повышенным риском оттока, оценивать потенциальные финансовые последствия потери клиентов и определять наиболее приоритетные объекты для retention-инвестиций.
#
### Ключевые бизнес-вопросы

В рамках данного раздела исследуются следующие вопросы:
-	Какие поведенческие факторы наиболее сильно связаны с будущим оттоком клиентов? 
-	Можно ли выявлять риск ухода до фактического снижения активности? 
-	Насколько точно модель способна прогнозировать churn? 
-	Какие признаки являются ранними индикаторами деградации клиентского поведения? 
-	Какие клиенты одновременно обладают: 
	  -	высоким риском ухода; 
	  -	высокой экономической ценностью; 
	  -	потенциалом успешной реактивации? 
-	Как приоритизировать retention-инвестиции с точки зрения ожидаемого бизнес-эффекта? 
#
### Бизнес-значимость

Практическая ценность данного раздела состоит в переходе от реактивного управления клиентской базой к проактивной retention-модели.
Вместо работы с уже потерянными клиентами компания получает возможность:
-	заранее обнаруживать формирующийся churn-risk; 
-	оценивать объём потенциально теряемой выручки; 
-	концентрировать CRM-ресурсы на наиболее ценных клиентах; 
-	снижать объём revenue-at-risk; 
-	повышать эффективность retention-бюджета. 

## 9.2 Analytical Approach

### Общая концепция анализа

В предыдущих разделах было показано, что переход клиентов в стадии At Risk и Churn сопровождается постепенным ухудшением покупательского поведения.
На основании этих наблюдений была построена система predictive customer analytics, ориентированная на раннее выявление пользователей с высокой вероятностью будущего оттока.
В отличие от традиционного RFM-подхода, модель учитывает не только текущее состояние клиента, но и динамику изменения его поведения во времени. 
#
### Логика исследования
Анализ построен как последовательность четырех взаимосвязанных этапов.

      Behavioral Signals
              ↓
      Churn Prediction
              ↓
      Business Risk Assessment
              ↓
      Retention Prioritization
#
### Analytical Framework
### 1. Churn Prediction Modeling  
На первом этапе строится модель прогнозирования вероятности оттока.  
Цель этапа — определить, способен ли набор поведенческих признаков заранее идентифицировать клиентов, которые в следующем периоде перейдут в стадии:
-	At Risk 
-	Churn

Для прогнозирования используется Random Forest Classifier.
#
### 2. Churn Driver Analysis  
После построения модели анализируются факторы, формирующие прогноз.  
Для интерпретации результатов используются:
-	Feature Importance; 
-	SHAP Analysis.

Данный этап позволяет понять:  
-	какие признаки наиболее влияют на churn probability; 
-	какие поведенческие изменения являются ранними сигналами риска; 
-	как именно модель принимает решения. 
#
### 3. Revenue-at-Risk Framework  
Высокая вероятность ухода не всегда означает высокий финансовый риск.
Поэтому следующим этапом является интеграция churn probability с показателями клиентской ценности.  

Цель этапа:  
-	определить объём потенциально теряемой выручки;  
-	выявить клиентов с максимальным business impact;  
-	перейти от churn management к revenue-at-risk management.  
#
### 4. Retention Prioritization Framework    
Финальный этап объединяет:   
-	churn probability; 
-	customer value; 
-	recovery probability.

Это позволяет определить клиентов, для которых удержание обладает максимальным ожидаемым экономическим эффектом.
Именно данный слой превращает predictive model в практический инструмент CRM-принятия решений. 
#
### Почему выбран Random Forest
Для решения задачи прогнозирования был выбран алгоритм Random Forest.
Причины выбора:  
- хорошо работает на нелинейных зависимостях;   
- устойчив к шуму;   
- автоматически учитывает взаимодействия признаков;   
- не требует сложной подготовки данных;   
- обеспечивает высокую интерпретируемость через Feature Importance и SHAP.

Дополнительным преимуществом является устойчивость модели к мультиколлинеарности и выбросам, что особенно важно для клиентских поведенческих данных. 
#
### Почему используется Time-Based Validation
Для оценки качества модели использован временной train-test split.
Обучение выполнялось на snapshot-периодах до февраля 2022 года, а тестирование — на следующем временном срезе.
Такой подход максимально приближен к реальному production-сценарию.
Модель всегда обучается на прошлом и прогнозирует будущее.
#
### Защита от Data Leakage
Отдельное внимание уделено предотвращению temporal leakage.
При построении признаков используются только данные, доступные на момент соответствующего snapshot.
Будущая информация не участвует в расчёте признаков и не влияет на обучение модели.
Это позволяет рассматривать полученные результаты как реалистичную оценку качества прогнозирования в реальном бизнес-процессе.

## 9.3 Feature Engineering

### Формирование признакового пространства

Для прогнозирования оттока был сформирован набор признаков, отражающих текущее состояние клиента, историю его взаимодействия с компанией и динамику изменения покупательского поведения. Все признаки рассчитываются на уровне snapshot и отражают информацию, доступную на момент прогнозирования.

### Блок 1. Current Behavioral State
Характеризует текущее состояние клиента.
Используемые признаки:

* *Recency* - Давность последней покупки  
* *Frequency* - Количество покупок  
* *Monetary* - Суммарная выручка  
* *Avg Check* - Средний чек  

Эти метрики отражают текущий уровень активности и коммерческой ценности клиента. 
#
### Блок 2. Purchase Cadence
Описывает покупательский ритм пользователя.

* *Avg Days Between Purchases* - Средний интервал между покупками

Данный показатель позволяет фиксировать замедление покупательской активности ещё до полного прекращения транзакций. 
#
### Блок 3. Behavioral Trend Features
Отражает изменение клиентского поведения относительно предыдущего периода.

* *Frequency Delta* - Изменение частоты покупок  
* *Monetary Delta* - Изменение выручки

Данные признаки позволяют выявлять ухудшение активности до наступления фактического churn.
#
### Блок 4. Growth Features
Используемые показатели:

* *Frequency Growth* - Темп изменения частоты покупок    
* *Monetary Growth* - Темп изменения выручки  

Именно данные признаки впоследствии оказываются среди наиболее сильных драйверов модели и позволяют фиксировать ранние признаки деградации клиентского поведения. 
#
#### *Figure 35. Структура признакового пространства*

				Current State
		(Recency, Frequency, Monetary)
		                +
				Purchase Cadence
		(Avg Days Between Purchases)
		                +
				Behavior Trends
		(Frequency Delta, Monetary Delta)
		                +
				Growth Metrics
		(Frequency Growth, Monetary Growth)
		                ↓
		        Churn Prediction

## 9.4 Model Development

### Постановка задачи прогнозирования

После анализа lifecycle-переходов и когортного поведения следующим шагом стало построение модели раннего выявления клиентов, находящихся на траектории деградации покупательской активности.  

Цель модели — прогнозировать вероятность ухудшения состояния клиента до фактического перехода в стадию оттока и предоставить основу для проактивного управления retention.  

В качестве целевого события был выбран переход клиента в одну из следующих lifecycle-стадий на следующем snapshot:
- At Risk  
- Churn
  
Таким образом, задача моделирования была сформулирована как задача бинарной классификации:   
Останется ли клиент в устойчивом состоянии или перейдет в зону риска в следующем периоде наблюдения?
#
### Архитектура обучающей выборки

Для каждого snapshot формировался набор поведенческих признаков клиента.
Модель обучалась на исторических snapshot-данных и прогнозировала состояние клиента в следующем временном периоде.  

Схема формирования данных:
				
						Snapshot T
						    ↓
						Customer Features
						    ↓
						Customer State at T+1
						    ↓
						Target Variable
		
Такой подход позволяет воспроизвести реальный production-сценарий использования модели, когда решение принимается исключительно на основе доступной исторической информации.
#
### Предотвращение Temporal Leakage
Одним из ключевых требований являлось исключение temporal leakage.   
Для этого использовался time-based split:
| Dataset |	  Период       |
|---------|----------------|
| Train   | до 2022-02-28  |
| Test	  | 2022-03-31     |

Модель обучалась только на исторических snapshots и тестировалась на более позднем временном периоде. Это обеспечивает корректную оценку способности модели обобщать знания на будущие данные.
#
### Выбор алгоритма
В качестве базового алгоритма был выбран Random Forest Classifier.  
Причины выбора:  
- способность моделировать нелинейные зависимости;  
- устойчивость к выбросам;  
- отсутствие требований к нормализации признаков;  
- хорошая интерпретируемость через Feature Importance и SHAP;  
- высокая устойчивость при работе с поведенческими признаками и ограниченным количеством snapshot-периодов.

Параметры модели:
- n_estimators = 500
- max_depth = 6
- min_samples_leaf = 30
- class_weight = balanced

## 9.9 Recovery Modeling & Retention Prioritization

### Business Question
Даже среди клиентов с высоким риском оттока существуют принципиальные различия.
Некоторые пользователи практически потеряны для бизнеса.
Другие могут быть успешно возвращены при своевременном вмешательстве.
Поэтому возникает вопрос:
> Какие клиенты одновременно обладают высоким риском оттока, высокой будущей ценностью и высокой вероятностью восстановления активности?
#
### Methodology
Для ранжирования клиентов сегмента At Risk по относительному потенциалу восстановления активности и последующей приоритизации retention-инвестиций был построен Recovery scoring framework.    

Целевая переменная:  
- Recovered = 1

если клиент в следующем периоде возвращается в:  
- Active   
- Loyal
  
состояние. 

Для оценки вероятности восстановления использовались:
- Frequency; 
- Monetary; 
- Avg Check; 
- Frequency Delta; 
- Monetary Delta; 
- Frequency Growth; 
- Monetary Growth; 
- Purchase Cadence.
  
После расчета Recovery Probability была сформирована итоговая метрика:  
- *Retention Priority Score*

		  Retention Priority  =  Churn Probability  ×  Expected Future Revenue  ×  Recovery Probability

Интегральный показатель Retention Priority Score позволил выделить группу клиентов, которые одновременно обладают:

#
### Business Implications
Главный вывод данного раздела состоит в том, что бизнесу необходимо управлять не churn probability, а объемом будущей выручки под риском.  

Показатель    

		Revenue-at-Risk = Expected Future Revenue × Churn Probability  

позволяет перейти от классического churn management к системе экономически обоснованной retention-приоритизации.    

Интеграция:  
- Churn Prediction; 
- Customer Value; 
- Recovery Modeling;
   
формирует полноценную систему data-driven retention management, ориентированную на сохранение будущих денежных потоков, а не просто на снижение количества ушедших клиентов. 

## 9.10 Retention Prioritization Score
### Business Question
Если ресурсов на удержание ограниченное количество, каких клиентов необходимо спасать в первую очередь?
Высокий риск оттока сам по себе не означает высокий бизнес-приоритет. Аналогично, высокая ценность клиента не гарантирует, что его возможно вернуть.
Поэтому финальной целью predictive framework является не просто прогнозирование churn, а определение клиентов, для которых retention-инвестиции способны принести максимальный экономический эффект.
#
### Analytical Approach
Для решения данной задачи был разработан интегральный показатель Retention Priority Score.  
Метрика объединяет три независимых компонента:  
- вероятность оттока (Churn Probability);   
- ожидаемую будущую ценность клиента (Expected Future Revenue);   
- вероятность успешного восстановления активности (Recovery Probability).
#  
### Формула
		Retention Priority Score =  Churn Probability  × Expected Future Revenue  × Recovery Probability  

Таким образом, высокий приоритет получают клиенты, которые одновременно:
- находятся под высоким риском потери; 
- обладают существенным future revenue potential; 
- сохраняют высокий потенциал восстановления.
  
Данный подход позволяет перейти от управления риском к управлению ожидаемой экономической отдачей от retention-интервенций.

При расширении охвата до 30% клиентов покрытие увеличивается до 81%, что указывает на высокую концентрацию ценности в верхней части скоринга и позволяет существенно снизить операционные затраты на retention-кампании.    

Интегральный показатель Retention Priority Score позволил выделить отдельную группу клиентов, сочетающих высокий churn-risk, значимый будущий revenue potential и высокую вероятность восстановления. Анализ показал, что наиболее приоритетными объектами удержания являются не только клиенты из сегмента At Risk, но и часть пользователей из Active и Loyal сегментов, демонстрирующих ранние признаки ухудшения поведения.   

Полученные результаты показывают, что риск оттока и экономическая значимость клиента не всегда совпадают. Значительная часть пользователей с высоким churn probability обладает ограниченной будущей ценностью, тогда как отдельные клиенты с умеренным риском формируют существенно больший объем потенциально сохраняемой выручки.
#
### Business Implications
Retention Prioritization Score переводит систему управления клиентской базой из режима:
*«Кто может уйти?»*
в режим:
*«Кого выгоднее всего сохранить?»*
Это позволяет существенно повысить эффективность CRM-бюджета за счет концентрации ресурсов на клиентах с максимальным ожидаемым возвратом инвестиций.

Высокий churn-risk не равен потерянному клиенту.

## 9.12 Key Findings & Business Implications
### Executive Summary
Построенная predictive-система показала, что риск оттока формируется задолго до фактической потери клиента и может быть выявлен по изменениям поведенческих характеристик пользователей. Наиболее сильными индикаторами будущей деградации выступают динамика покупательской частоты, изменение выручки и увеличение интервалов между покупками. 
Дополнительный анализ показал, что экономический риск бизнеса определяется не количеством клиентов с высокой вероятностью ухода, а объемом будущей выручки, находящейся под угрозой потери. Это делает Revenue-at-Risk более полезным инструментом управления, чем традиционный churn monitoring. 
Интеграция Recovery Modeling позволила расширить predictive framework от прогнозирования оттока к прогнозированию эффективности удержания. В результате была сформирована система retention-prioritization, способная определять клиентов с максимальной ожидаемой отдачей от CRM-инвестиций. 
#
### Key Insights
### 1. Поведенческие тренды важнее статических метрик   
Наиболее значимыми факторами риска являются изменения покупательского поведения, а не только текущее состояние клиента.
Ухудшение Frequency Growth, Monetary Growth и Purchase Cadence выступает ранним сигналом будущего churn. 

### 2. Revenue-at-Risk важнее количества рискованных клиентов     
Не все клиенты с высоким churn probability одинаково опасны для бизнеса.
Основной риск создают пользователи, сочетающие высокий риск ухода и высокую будущую ценность. 

### 3. Не каждый рискованный клиент требует удержания      
Recovery Modeling показал, что вероятность возврата существенно различается между пользователями.
Часть клиентов остается экономически целесообразной целью для retention-инвестиций даже при высоком риске ухода. 

### 4. Максимальную ценность создает сегмент High Value × High Risk × High Recovery      
Именно эта группа обеспечивает наибольший потенциальный эффект от CRM-вмешательства и должна являться главным объектом персонализированных retention-кампаний. 
#
### Strategic Implications
Проведенный анализ показывает необходимость перехода от Mass CRM к Predictive Retention Management и далее от Risk-Based Retention к Value-Adjusted Retention, где решение об удержании принимается одновременно на основе:  
- риска потери клиента;   
- ожидаемой будущей выручки;   
- вероятности успешного восстановления.   
#
### Business Impact
Внедрение подобного подхода позволяет:
- повысить ROI retention-кампаний; 
- снизить неэффективные CRM-расходы; 
- сфокусировать бюджет на наиболее ценных клиентах; 
- увеличить объем сохраненной выручки; 
- перейти от churn management к Revenue-at-Risk management. 
#
### Финальный Executive Takeaway
Максимальную ценность для бизнеса представляет не выявление клиентов с высокой вероятностью ухода, а выявление клиентов, чья потеря создаёт наибольший финансовый ущерб и при этом остаётся предотвратимой. Объединение churn prediction, customer value и recovery modeling формирует основу для перехода от массовых CRM-коммуникаций к системе data-driven retention management. 

		Churn Prediction
		        ↓
		Revenue-at-Risk
		        ↓
		Recovery Modeling
		        ↓
		Retention Prioritization
		        ↓
		Data-Driven CRM Decision System

- Какие сегменты обладают наибольшим потенциалом возврата retention-инвестиций?

## 10.4 Lifecycle Stability Analysis
### Цель анализа
Retention, churn и revenue retention показывают результат происходящих процессов.   
Однако они не отвечают на вопрос:  
> Насколько стабильна сама структура клиентской базы?  

Например:  
- retention может снижаться; 
- churn может расти; 
- но при этом большая часть клиентов может оставаться внутри своих lifecycle-сегментов.
  
И наоборот:  
- общий retention может выглядеть приемлемо; 
- но клиенты могут активно мигрировать между стадиями,

что указывает на скрытую нестабильность клиентского портфеля.   
Для оценки структурной устойчивости lifecycle-системы были дополнительно рассчитаны два интегральных показателя стабильности.
#
### Методология
#### • Lifecycle Stability Index
Показывает долю клиентов, сохранивших свою lifecycle-стадию между соседними snapshot-периодами.  

Формула:

		Lifecycle Stability Index  =  Customers with unchanged stage  /  Total customers

Высокое значение индекса означает:    
- низкую турбулентность клиентской базы;   
- стабильность lifecycle-структуры;   
- предсказуемость поведения пользователей.   

#### • Healthy Stability Index
Дополнительно был рассчитан показатель устойчивости здоровой части клиентской базы.  
В расчет включаются только стадии:    
- Active   
- Loyal
  
Формула:  

		Healthy Stability Index  =  Healthy customers remaining healthy  /  Total healthy customers

Показатель позволяет оценить способность бизнеса удерживать наиболее ценных клиентов внутри активной части lifecycle.
#
### Reference Visualization
См. [Figure 50](#fig50) Динамика Lifecycle Stability Index и Healthy Stability Index между последовательными snapshot-периодами.

#
### Результаты анализа
#### Общая стабильность клиентской базы
Анализ показал, что по мере развития жизненного цикла структура клиентской базы становится менее устойчивой.
Несмотря на наличие значительного сегмента клиентов, сохраняющих свою стадию между периодами, наблюдается постепенное увеличение миграции между lifecycle-состояниями.  
Это согласуется с результатами предыдущих разделов:   
- retention постепенно снижается;   
- churn растет;   
- увеличивается количество клиентов, переходящих в At Risk.
#
#### Устойчивость здоровой части базы
Особый интерес представляет динамика Healthy Stability Index.
Показатель отражает способность компании удерживать клиентов внутри наиболее ценных сегментов:  
- Active;   
- Loyal.
  
Снижение данного индекса означает, что даже сохраненные клиенты начинают постепенно деградировать внутри жизненного цикла и переходить в более рискованные состояния.
Это особенно важно с точки зрения долгосрочной экономики бизнеса, поскольку именно данные сегменты формируют основную часть будущей выручки.
#
### Бизнес-интерпретация
Наблюдаемый разрыв между Healthy Stability Index и Lifecycle Stability Index отражает запас структурной устойчивости клиентской базы. Доминирование Healthy Stability Index на протяжении всего периода анализа указывает на локальный характер деградации, затрагивающей преимущественно периферийные lifecycle-сегменты. Это делает Lifecycle Stability Index опережающим индикатором структурных изменений: сокращение разрыва между индексами будет означать, что ухудшение начинает распространяться на наиболее ценных клиентов, создавая риск системного снижения retention, роста churn и долгосрочной потери выручки.

Таким образом Lifecycle Stability Index выступает ранним индикатором ухудшения customer health.
В отличие от churn, который фиксирует уже состоявшуюся потерю клиента, снижение стабильности позволяет обнаруживать проблемы значительно раньше:

			Снижение стабильности
			          ↓
			Рост миграции между стадиями
			          ↓
			Рост At Risk сегмента
			          ↓
			     Рост churn
			          ↓
			    Потеря выручки
				
Поэтому стабильность lifecycle-структуры может использоваться как leading indicator будущих проблем удержания.

## 10.6 Revenue-at-Risk
### Цель анализа

После определения сегментов, формирующих выручку, необходимо понять, какая часть этой выручки находится под угрозой потери.
Высокая выручка сама по себе не гарантирует устойчивость бизнеса.

Для принятия управленческих решений важно учитывать вероятность потери клиентов и потенциальные финансовые последствия такого оттока.
С этой целью была построена система оценки Revenue-at-Risk.
#
### Методология

Для каждого lifecycle-сегмента рассчитывается показатель:

			Revenue At Risk  =  Revenue  ×  Churn Probability

Метрика объединяет:  
- текущую экономическую ценность сегмента;  
- вероятность его потери;
- потенциальный финансовый ущерб.

В отличие от обычного churn-risk данный показатель позволяет оценивать риск в денежном выражении.
#
### Результаты анализа

| Stage   | Revenue-at-Risk |
| ------- | --------------: |
| Active  |           1.39M |
| At Risk |           1.35M |
| Loyal   |           1.02M |
| New     |           0.25M |
| Churn   |           0.00M |

Полученные результаты приводят к неожиданному выводу.
Интуитивно можно было ожидать, что максимальный риск будет сосредоточен внутри сегмента At Risk.
Однако на практике наибольший объём потенциальных потерь формируется внутри Active-клиентов.
#
### Почему Active оказался опаснее At Risk

Причина заключается в сочетании факторов:

***Active***    
- крупнейший сегмент базы;  
- значительный объём выручки;  
- уже заметный churn-risk.  
	
***At Risk***    
- более высокий риск;  
- меньшая выручка.  

В результате общий объём потенциальных потерь оказывается сопоставимым:

| Segment | Revenue-at-Risk |
| ------- | --------------: |
| Active  |           1.39M |
| At Risk |           1.35M |

Это означает, что значительная часть будущего churn формируется ещё до того, как клиент официально попадает в At Risk.
#
### Loyal Segment Risk     
Особенно важен результат для Loyal-сегмента.   
- Несмотря на минимальную вероятность оттока: Average Churn Risk = 0.16    
- высокая концентрация выручки приводит к тому, что потенциальные потери достигают: 1.02M  

Таким образом, даже небольшое ухудшение удержания наиболее ценных клиентов может иметь существенные финансовые последствия.
#
### Бизнес-интерпретация

Анализ показывает существование трёх разных типов риска.

**Active Risk**

- Риск ранней деградации клиентской базы.  
- Клиенты ещё не выглядят проблемными, но уже формируют значительный объём будущих потерь.  

**At Risk Risk**

- Риск непосредственного перехода в churn.  
- Наиболее очевидная зона удержания.  

**Loyal Risk**

- Риск потери высокоценной выручки.  
- Даже небольшой churn внутри данного сегмента оказывает непропорционально сильное влияние на бизнес.  
#
### Управленческие выводы

## 10.7 Retention Prioritization
### Цель анализа

Предыдущие подразделы позволили определить:
- какие сегменты формируют основную выручку;
- где сосредоточен churn-risk;
- какие группы клиентов создают наибольший объём revenue-at-risk.

Однако ограниченность CRM-ресурсов не позволяет одинаково активно работать со всеми клиентами одновременно.
Поэтому следующим шагом является определение приоритетности retention-инвестиций между lifecycle-сегментами.

Цель анализа — определить, какие группы клиентов способны обеспечить наибольший экономический эффект от удержания.
#
### Методология

Для ранжирования сегментов используется интегральный показатель Priority Score.

Показатель объединяет:
- размер сегмента;
- объём выручки;
- churn-risk;
- величину revenue-at-risk.

Чем выше значение Priority Score, тем выше ожидаемый бизнес-эффект от retention-интервенций.
#
### Результаты анализа

| Lifecycle Stage | Priority Score |
| --------------- | -------------: |
| Active          |           11.9 |
| At Risk         |           11.5 |
| Loyal           |            8.8 |
| New             |            2.1 |
| Churn           |            0.0 |

Полученные результаты показывают, что наиболее приоритетным сегментом неожиданно оказывается не At Risk, а Active.
#
### Почему Active занимает первое место

На первый взгляд At Risk обладает более высоким риском потери. Однако Priority Score учитывает не только вероятность оттока, но и потенциальный масштаб последствий.

Active-клиенты одновременно характеризуются:  
- крупнейшим размером сегмента;    
- значительным объёмом выручки;  
- заметным уровнем churn-risk.  

Поэтому даже небольшое ухудшение удержания внутри Active-сегмента способно привести к существенным потерям будущей выручки.
#
### Роль Loyal-сегмента

Отдельного внимания заслуживает Loyal-сегмент. Несмотря на низкий churn-risk, высокая концентрация выручки приводит к тому, что данный сегмент сохраняет высокий приоритет удержания. Потеря одного Loyal-клиента обходится бизнесу существенно дороже, чем потеря среднего пользователя базы. Поэтому задача удержания Loyal-клиентов заключается не в массовых реактивационных кампаниях, а в предотвращении деградации наиболее ценной части клиентского портфеля.
#
### Бизнес-интерпретация

Анализ показывает, что классическая модель:

			At Risk → CRM Action

слишком упрощает реальную структуру риска.

На практике наиболее эффективная стратегия должна охватывать три разные зоны:

- **Active**  
Предотвращение будущей деградации.

- **At Risk**  
Возврат клиентов до перехода в churn.

- **Loyal**  
Защита наиболее ценной части выручки.

## 10.9 Key Findings & Business Implications
### Executive Summary

Разработанный Lifecycle KPI Framework показал, что клиентская база аптечной сети демонстрирует признаки постепенного ухудшения customer health по мере развития жизненного цикла.

Несмотря на наличие устойчивого сегмента лояльных покупателей, наблюдается последовательное снижение retention, рост churn и увеличение объема revenue-at-risk внутри активной части клиентской базы.

Одновременно анализ выявил высокую концентрацию выручки среди ограниченного числа наиболее ценных клиентов. Сегмент Loyal составляет менее 16% клиентской базы, однако генерирует более половины всей выручки компании.

Дополнительный анализ lifecycle stability показал снижение устойчивости структуры клиентского портфеля. Увеличение миграции между lifecycle-сегментами происходит раньше фактического churn и может рассматриваться как ранний индикатор ухудшения customer health.

В совокупности результаты показывают, что ключевой риск бизнеса связан не только с уже потерянными клиентами, но и с постепенной деградацией активной части клиентской базы, формирующей будущую выручку компании.
#
### Key Insights

### 1. Клиентская база демонстрирует ухудшение customer health

Lifecycle KPI Trends показывают последовательное снижение удержания клиентов на протяжении периода наблюдения.

Retention Rate снижается: 59.9% → 47.3%  
Одновременно Churn Rate возрастает: 5.4% → 25.2%  

Несмотря на наличие реактивации пользователей, скорость возврата клиентов недостаточна для компенсации растущего оттока.
Это свидетельствует об ухудшении общего состояния клиентского портфеля.
#
### 2. Структурная устойчивость клиентской базы снижается раньше фактического churn

Анализ Lifecycle Stability показал, что деградация клиентской базы начинается значительно раньше потери клиентов.
Рост количества переходов между lifecycle-сегментами сопровождается снижением доли клиентов, сохраняющих устойчивое поведение между периодами наблюдения.

Особенно важным является ухудшение Healthy Stability Index, отражающего устойчивость сегментов Active и Loyal.
Это означает, что ухудшение customer health происходит внутри активной клиентской базы задолго до появления явного churn.
#
### 3. Выручка бизнеса крайне концентрирована

Распределение выручки между lifecycle-сегментами оказалось существенно более неравномерным, чем распределение клиентов.

Сегмент Loyal:  
- 15.8% клиентов;  
- 55.4% общей выручки.  

Средний Revenue Index для Loyal-клиентов достигает 3.51, что более чем в три раза превышает средний уровень по базе.

Таким образом, устойчивость бизнеса критически зависит от сохранения ограниченного сегмента высокоценных покупателей.
#
### 4. Основной финансовый риск сосредоточен внутри Active и At Risk сегментов

Расчет Revenue-at-Risk показал, что наибольший объем потенциальных потерь связан не только с клиентами, уже находящимися в стадии риска.  

Наиболее высокий уровень Revenue-at-Risk наблюдается для:

| Stage   | Revenue-at-Risk |
| ------- | --------------: |
| Active  |           1.39M |
| At Risk |           1.35M |
| Loyal   |           1.02M |

Полученные результаты показывают, что значительная часть будущих потерь формируется еще до перехода клиента в стадию At Risk.
Это делает Active-сегмент критически важной зоной раннего retention-вмешательства.
#
### 5. Contraction Revenue становится системной угрозой будущему росту

Анализ Revenue Dynamics показал постепенное ухудшение структуры удержанной выручки.
В начале периода Expansion Revenue превышает Contraction Revenue.   

Однако к концу периода ситуация меняется:

| Metric              |  Value |
| ------------------- | -----: |
| Expansion Revenue   | ~1.06M |
| Contraction Revenue | ~1.49M |


Рост сокращения потребления внутри существующей клиентской базы начинает превышать рост потребления.
Это указывает на накопление будущих потерь еще до появления полноценного churn.
#
### 6. Приоритет удержания должен смещаться от реактивной к проактивной модели

Retention Prioritization Framework показал следующую структуру приоритетов:

| Segment | Priority Score |
| ------- | -------------: |
| Active  |           11.9 |
| At Risk |           11.5 |
| Loyal   |            8.8 |
| New     |            2.1 |

Наиболее высокий ожидаемый эффект от retention-инвестиций сосредоточен внутри Active-сегмента.

Это означает, что эффективная стратегия удержания должна начинаться значительно раньше появления явных признаков оттока.
#
### Business Interpretation

Объединение результатов всех аналитических блоков позволяет описать текущее состояние клиентской базы следующим образом.
Компания обладает относительно небольшим, но экономически крайне ценным сегментом лояльных клиентов, который обеспечивает основную часть выручки бизнеса.

Однако по мере развития жизненного цикла наблюдаются признаки постепенного ухудшения customer health:  
- снижается retention;  
- растет churn;  
- ухудшается lifecycle stability;  
- увеличивается contraction revenue;  
- растет объем revenue-at-risk.  

Наиболее важным выводом является то, что основная зона риска находится не среди уже уходящих клиентов, а внутри Active-сегмента, который формирует будущий Loyal-сегмент и будущую выручку компании.

Фактически бизнес сталкивается не столько с проблемой возврата потерянных клиентов, сколько с проблемой предотвращения деградации активной части клиентской базы.
#
### Business Impact

Полученные результаты показывают, что дальнейший рост бизнеса будет определяться способностью компании решать три ключевые задачи:

**Сохранение высокоценного Loyal-сегмента**  
Минимизация потерь наиболее прибыльных клиентов.

**Раннее выявление деградации Active-клиентов**  
Предотвращение перехода пользователей в стадию риска.

Управление будущими потерями до возникновения фактического churn.

# 12. Integrated Business Insights
## 12.1 Synthesis of Findings

### Интеграция результатов проекта
Проведённый анализ позволил исследовать клиентскую базу аптечной сети с четырёх взаимодополняющих точек зрения:  
1.	Lifecycle Analytics — динамика переходов между состояниями жизненного цикла. 
2.	Cohort Analytics — долгосрочное удержание и накопление клиентской ценности. 
3.	Predictive Analytics — раннее выявление риска оттока. 
4.	KPI Framework — мониторинг состояния клиентского портфеля и концентрации финансовых рисков.
   
Несмотря на различие аналитических подходов, все разделы приводят к одному и тому же системному выводу:  
> Основной риск бизнеса связан не с монетизацией существующих клиентов, а с неспособностью переводить новых покупателей в сегмент долгосрочно удерживаемой аудитории.
#
### Инсайт 1. Основная потеря клиентов происходит сразу после первой покупки
Результаты cohort-анализа показали, что уже в первые месяцы жизненного цикла происходит наиболее интенсивный отток пользователей.
Средний User Retention снижается примерно до 26% уже после первых месяцев взаимодействия с компанией.    

Lifecycle-анализ подтверждает данный вывод:  
- большинство клиентов начинают деградировать ещё до перехода в формальный churn;  
- поток клиентов из Active в At Risk стабильно превышает обратные переходы;   
- churn является следствием более ранних процессов ухудшения customer health.   

Таким образом, ключевая проблема бизнеса находится не в удержании лояльных клиентов, а значительно раньше — на этапе формирования повторной покупки.
#
### Инсайт 2. После прохождения раннего периода формируется устойчивое ядро клиентов
Несмотря на высокий initial churn, retention-кривые демонстрируют выраженное плато после первых месяцев жизненного цикла.
Retention decay существенно замедляется, а большая часть оставшихся пользователей сохраняет стабильную активность на протяжении длительного периода времени.  

Дополнительно это подтверждается результатами lifecycle stability analysis:   
- Lifecycle Stability Index достигает 67.7%;  
- Healthy Stability Index составляет 71.0%.
  
Это означает, что после прохождения критической стадии ранней селекции клиентская база становится существенно более устойчивой.
Для бизнеса это особенно важно:
каждый дополнительный клиент, удержанный в первые месяцы после первой покупки, получает значительно более высокую вероятность долгосрочного сохранения.
#
### Инсайт 3. Выручка удерживается лучше, чем пользователи
Практически во всех когортах Revenue Retention систематически превышает User Retention.
Даже при сокращении клиентской базы компания сохраняет около 79% выручки между соседними периодами.  

Это свидетельствует о наличии механизма естественной селекции:  
- менее вовлечённые клиенты покидают продукт быстрее;    
- наиболее ценные пользователи остаются значительно дольше;   
- концентрация выручки постепенно возрастает внутри удержанной аудитории.   

Фактически система теряет клиентов быстрее, чем теряет деньги.  
#
### Инсайт 4. Ценность клиента растёт по мере развития жизненного цикла
Анализ Revenue per Retained User показал устойчивый рост монетизации удержанных пользователей.
В наиболее зрелых когортах средняя выручка на одного retained customer превышает стартовый уровень на 43%.
Одновременно накопленный LTV продолжает увеличиваться на протяжении всего наблюдаемого горизонта и достигает 6.3–6.8 тыс. денежных единиц.
Это означает, что долгосрочная ценность клиента формируется не в момент первой покупки, а в процессе его постепенного развития внутри системы.
Каждый пользователь, успешно прошедший ранний этап жизненного цикла, становится всё более экономически значимым для бизнеса.
#
### Инсайт 5. Основной финансовый риск сосредоточен внутри Active-сегмента
Predictive Analytics и Revenue-at-Risk анализ выявили важный управленческий вывод.
Наиболее крупный объём потенциальных потерь формируется не среди уже проблемных клиентов, а среди пользователей, которые ещё остаются активными.  

Revenue-at-Risk:

Это означает, что деградация клиентского поведения начинается значительно раньше момента явного ухода клиента.
Следовательно, эффективная retention-стратегия должна работать проактивно, а не реактивно.
#
### Инсайт 6. Бизнес становится зависимым от ограниченного числа лояльных клиентов
Несмотря на то что сегмент Loyal составляет лишь около 16% клиентской базы, он генерирует 55.4% всей выручки компании. Это создаёт одновременно два эффекта:  

Позитивный:  
- высокая устойчивость денежного потока;   
- сильная монетизация удержанных клиентов.     

Негативный:  
- возрастающая концентрация бизнеса вокруг ограниченного числа покупателей;  
- риск ускоренного ухудшения выручки при деградации loyal-сегмента.
  
Именно поэтому сохранение здоровья данной группы становится критически важным элементом долгосрочной стратегии роста.

Следовательно:

## 12.2 Unified Customer Lifecycle Model
### Единая модель формирования клиентской ценности   
На основании результатов всех разделов была сформирована интегрированная модель жизненного цикла клиента, отражающая полный путь создания экономической ценности внутри клиентской базы.  
Модель показывает, каким образом отдельные этапы клиентского поведения формируют долгосрочные бизнес-результаты компании.
#
### *Этап 1. Acquisition*
Жизненный цикл начинается с привлечения нового клиента. На данном этапе пользователь совершает первую покупку и попадает в сегмент New. Однако первая транзакция сама по себе практически не создаёт долгосрочной ценности. Основной вопрос этого этапа: 
> Сможет ли клиент совершить повторную покупку?  
#
### *Этап 2. Early Retention*
Следующий этап является наиболее критичным для всей системы. Именно здесь происходит основной объём потерь клиентской базы. Большинство пользователей не переходят к регулярному взаимодействию и постепенно покидают продукт. 
Данный этап определяет:    
- будущий retention;   
- будущий revenue retention;   
- будущий LTV.   

Именно здесь сосредоточен основной резерв роста бизнеса.  
#
### *Этап 3. Behavioral Selection*
Пользователи, успешно преодолевшие ранний период, начинают демонстрировать более устойчивое поведение. Происходит естественная селекция клиентской базы:    
- случайные покупатели уходят;    
- остаются более вовлечённые пользователи.   
  
На этом этапе начинают формироваться будущие loyal customers. 
#
### *Этап 4. Loyalty Formation*
После прохождения стадии селекции клиент переходит в фазу устойчивого потребления.    
Характерные признаки этапа:    
- высокая частота покупок;     
- низкий churn-risk;     
- высокая вероятность сохранения активности.    

Именно здесь начинает концентрироваться большая часть выручки бизнеса.
#
### *Этап 5. Customer Value Expansion*
По мере развития жизненного цикла удержанные клиенты постепенно увеличивают свою экономическую ценность. Наблюдается:   
- рост RPRU;  
- рост cumulative revenue;    
- накопление LTV.   

На данном этапе каждый дополнительный месяц удержания приносит всё больший экономический эффект.
#
### *Этап 6. Revenue Concentration*
В результате долгосрочного удержания происходит концентрация бизнеса вокруг наиболее ценных клиентов. Сегмент Loyal становится главным источником выручки компании. Формируется устойчивая monetization engine всего бизнеса.  
#
### *Этап 7. Customer Health Management*
Заключительный этап представляет собой постоянный мониторинг состояния клиентского портфеля. Для управления системой используются:    
- Lifecycle KPI Framework;     
- Predictive Churn Model;     
- Revenue-at-Risk Monitoring;    
- Retention Prioritization Framework.   

Именно здесь аналитика превращается в инструмент принятия решений.

## 12.3 Strategic Priorities
### От аналитических выводов к бизнес-действиям
Проведенный анализ показал, что ключевые ограничения роста аптечной сети связаны не с привлечением новых клиентов и не с монетизацией удержанной аудитории, а с высокой потерей пользователей на ранних этапах жизненного цикла.
Несмотря на наличие устойчивого сегмента лояльных клиентов, формирующего более половины выручки компании, значительная часть покупателей не переходит к повторному потреблению после первой покупки. В результате бизнес вынужден постоянно компенсировать потери клиентской базы за счет новых привлечений, что повышает зависимость от маркетинговых инвестиций и ограничивает долгосрочный рост клиентской ценности.
Полученные результаты позволяют сформулировать четыре ключевых стратегических приоритета развития retention-системы.
#
### Приоритет 1. Снижение раннего оттока после первой покупки
Наиболее критическим ограничением текущей lifecycle-модели является резкий churn в первые месяцы после привлечения клиента.
Когортный анализ показал, что уже ко второму месяцу жизненного цикла удержание снижается примерно до 26%, после чего скорость оттока существенно замедляется и формируется стабильное ядро постоянных покупателей.
Это означает, что основная часть будущих потерь LTV возникает еще до формирования регулярного покупательского поведения.   

Основными направлениями развития должны стать:  
- стимулирование второй покупки;  
- персонализированные post-purchase коммуникации;   
- механики раннего вовлечения клиентов;   
- программы ускоренного формирования покупательской привычки.   

Даже умеренное улучшение раннего retention способно оказать непропорционально сильное влияние на долгосрочную выручку за счет увеличения числа клиентов, достигающих стадии высокой монетизации.
#
### Приоритет 2. Защита ядра высокоценных клиентов
Lifecycle-анализ показал высокую концентрацию выручки внутри сегмента Loyal.
Несмотря на относительно небольшой размер сегмента, лояльные клиенты формируют около 55% общей выручки компании и демонстрируют наиболее высокий уровень покупательской активности.
Дополнительный риск заключается в том, что значительная часть долгосрочного LTV бизнеса зависит именно от сохранения данной аудитории.  

Для повышения устойчивости выручки необходимо:  
- внедрять программы лояльности;  
- использовать персонализированные CRM-сценарии; 
- развивать механики удержания high-value клиентов;   
- регулярно мониторить показатели customer health и churn risk внутри Loyal-сегмента.  

Сохранение ядра лояльных покупателей обеспечивает максимальную защиту будущих денежных потоков компании.
#
### Приоритет 3. Проактивное управление Revenue-at-Risk
Интеграция predictive analytics показала, что наибольший объем потенциальных потерь сосредоточен не среди уже ушедших клиентов, а среди сегментов Active и At Risk. Совокупный объем Revenue-at-Risk внутри этих групп превышает 2.7 млн денежных единиц и существенно превосходит аналогичные показатели остальных lifecycle-сегментов.
Данный результат указывает на необходимость перехода от реактивной модели удержания к проактивному управлению риском.  

Приоритетными направлениями являются:
- регулярное обновление churn-scoring модели; 
- мониторинг динамики вероятности оттока; 
- раннее выявление признаков деградации поведения; 
- автоматизация retention-интервенций для клиентов с повышенным риском ухода. 

Такой подход позволяет предотвращать потери до фактического прекращения активности пользователя.
#
### Приоритет 4. Развитие системы Customer Health Monitoring
Отдельные аналитические исследования дают понимание причин потерь клиентской базы, однако для практического управления retention необходим постоянный мониторинг состояния клиентского портфеля.  
Разработанный Lifecycle KPI Framework создает основу для построения полноценной системы Customer Health Monitoring, объединяющей:  
- lifecycle metrics; 
- retention metrics; 
- revenue metrics; 
- churn-risk indicators; 
- revenue-at-risk indicators; 
- retention prioritization metrics. 

Особую ценность представляет возможность отслеживания изменений клиентского здоровья во времени через показатели:
- Retention Rate;  
- Revenue Retention; 
- Lifecycle Stability Index; 
- Healthy Stability Index; 
- Revenue-at-Risk; 
- Priority Score. 

Это позволяет превратить результаты анализа в постоянно действующую систему поддержки управленческих решений.

Полученные результаты показывают, что клиентская база характеризуется выраженной двухфазной структурой жизненного цикла.
На первом этапе происходит интенсивный отток пользователей после первой покупки. Большая часть клиентов не формирует устойчивого покупательского поведения и прекращает взаимодействие с компанией в течение первых месяцев после привлечения.
На втором этапе формируется относительно небольшое, но стабильное ядро лояльных покупателей, которое обеспечивает основную часть долгосрочной выручки бизнеса.

Когортный анализ подтвердил, что именно раннее удержание является главным фактором формирования долгосрочного LTV. Несмотря на существенное сокращение клиентской базы, удержанные пользователи демонстрируют устойчивый рост экономической ценности, что проявляется в более высокой динамике Revenue Retention по сравнению с User Retention и росте Revenue per Retained User на поздних этапах жизненного цикла.
Дополнительный анализ переходов между lifecycle-сегментами показал постепенное ухудшение customer health по мере старения клиентской базы. Снижение retention, рост churn и превышение contraction revenue над expansion revenue свидетельствуют о накоплении будущих потерь внутри существующего клиентского портфеля.
Разработанная predictive churn-модель продемонстрировала высокое качество прогнозирования и позволила перейти от ретроспективного анализа к раннему выявлению клиентов с повышенной вероятностью ухода. Интеграция прогнозного риска с показателями клиентской ценности позволила сформировать систему приоритизации retention-инвестиций и количественно оценить объем потенциальных финансовых потерь.

На основе результатов проекта был построен Lifecycle KPI Framework, объединяющий показатели удержания, монетизации, клиентского здоровья и прогнозного риска в единую систему мониторинга. Такой подход позволяет не только объяснять происходящие изменения клиентской базы, но и управлять ими в режиме непрерывного наблюдения. В целом результаты исследования показывают, что основной потенциал роста бизнеса находится не в расширении объема привлечения клиентов и не в дополнительной монетизации существующей лояльной аудитории. Наибольший экономический эффект может быть достигнут за счет улучшения раннего удержания, увеличения доли клиентов, переходящих в сегмент регулярных покупателей, и более эффективного управления Revenue-at-Risk.
Именно развитие retention-системы, основанной на данных и прогнозной аналитике, представляет собой наиболее перспективное направление повышения долгосрочной клиентской ценности, устойчивости выручки и эффективности инвестиций в развитие клиентской базы.

# 13. Strategic Recommendations
## 13.1 Retention Strategy
### Strategic Objective

Основной целью retention-стратегии является снижение объема потенциально теряемой выручки за счет раннего выявления клиентов с высоким риском оттока и концентрации ресурсов на пользователях с максимальным ожидаемым эффектом удержания.

Результаты predictive analytics показали, что риск потери клиентов распределяется неравномерно внутри клиентской базы. Наибольший объем потенциальных потерь сосредоточен в сегментах Active и At Risk, которые одновременно характеризуются высокой концентрацией revenue-at-risk и максимальными значениями Retention Priority Score.

Дополнительно анализ факторов оттока показал, что ухудшение клиентского поведения начинается задолго до фактического перехода пользователя в стадию Churn. Наиболее сильными предикторами будущего оттока являются снижение покупательской активности, ухудшение динамики выручки и увеличение интервалов между покупками.
#
### Key Findings Driving the Strategy
***Revenue-at-Risk Concentration***

Основная часть потенциальных потерь выручки сосредоточена в двух сегментах:

| Segment | Revenue-at-Risk |
| ------- | --------------: |
| Active  |           1.39M |
| At Risk |           1.35M |

Совокупно данные сегменты формируют более 2.7 млн потенциально уязвимой выручки. Это означает, что значительная часть будущих потерь возникает еще до формального перехода клиента в состояние оттока.
#
***Main Churn Drivers***

SHAP-анализ и Feature Importance показали, что наиболее важными индикаторами будущего ухода являются:

| Driver                     | Importance |
| -------------------------- | ---------: |
| Frequency                  |      22.4% |
| Monetary Growth            |      18.5% |
| Monetary Delta             |      14.0% |
| Frequency Growth           |      12.5% |
| Avg Days Between Purchases |      10.4% |
| Monetary                   |       9.2% |

Полученные результаты показывают, что ключевым сигналом риска является не текущее значение клиентских метрик, а ухудшение их динамики во времени.
Особенно важным ранним индикатором выступает снижение частоты покупок и замедление темпов потребления.
#
### Strategic Recommendation 1
*Создать систему раннего предупреждения и удержания.*

Результаты модели подтверждают возможность обнаружения признаков деградации клиентского поведения до фактического ухода клиента.
Вместо запуска реактивных кампаний после перехода пользователя в At Risk компания должна использовать churn probability как систему раннего предупреждения.
Клиенты с растущим интервалом между покупками, снижением Frequency Growth и ухудшением Monetary Growth должны автоматически попадать в retention-monitoring до появления критического риска.

**Expected Business Effect**  
- снижение объема предотвращаемого churn;  
- более раннее удержание клиентов;  
- сокращение затрат на реактивацию.  
#
### Strategic Recommendation 2
*Приоритизировать клиентов по ожидаемому влиянию на бизнес.*

Анализ показал, что высокий churn-risk сам по себе не является достаточным основанием для retention-инвестиций.
Для оценки экономической эффективности удержания был разработан интегральный показатель Retention Priority Score, который объединяет:  
- вероятность оттока;  
- ожидаемую будущую выручку;  
- вероятность успешной реактивации.  

Данный подход позволяет отказаться от массовых кампаний и концентрировать ресурсы на клиентах с максимальным ожидаемым ROI удержания.

**Expected Business Effect**  
- рост эффективности CRM-коммуникаций;
- снижение стоимости удержания;
- повышение возврата инвестиций в retention.
#
### Strategic Recommendation 3
*Разработать программы удержания, ориентированные на восстановление клиентов.*

Recovery-анализ показал, что часть клиентов из At Risk сегмента обладает высокой вероятностью возвращения в активное состояние. При этом наиболее перспективные клиенты одновременно характеризуются:  
- высоким churn-risk;  
- значимой ожидаемой ценностью;  
- высоким recovery potential.  

Для таких пользователей рекомендуется разработать отдельные сценарии реактивации:  
- персонализированные предложения;  
- целевые бонусные программы;  
- программы возврата клиентов;  
- коммуникации на основе жизненного цикла.
  
**Expected Business Effect**  
- повышение эффективности реактивации;  
- восстановление ранее потерянной выручки;  
- снижение долгосрочного churn.  
#
### Strategic Recommendation 4
*Фокус на деградации поведения, а не на Lifecycle стадиях.*

Одним из наиболее важных результатов анализа стало то, что признаки будущего оттока появляются раньше, чем клиент меняет lifecycle-сегмент. Поэтому CRM-система должна отслеживать не только текущую lifecycle-stage клиента, но и изменения его поведения:  
- снижение Frequency;   
- ухудшение Monetary Dynamics;  
- увеличение Purchase Gap;  
- снижение Frequency Growth.  

Именно данные сигналы должны выступать триггерами для retention-интервенций.
#
### Strategic Implications

Построенная система predictive retention позволяет компании перейти от реактивного управления оттоком к проактивной модели удержания клиентов.
Основное внимание должно быть сосредоточено не на уже потерянных клиентах, а на раннем выявлении признаков деградации поведения и адресном распределении retention-ресурсов между наиболее ценными и потенциально восстанавливаемыми пользователями.

## 13.2 Lifecycle Management Strategy
### Strategic Objective

Цель lifecycle-стратегии заключается в увеличении доли клиентов в наиболее ценных и устойчивых состояниях жизненного цикла, а также в сокращении скорости деградации клиентской базы.

Результаты lifecycle analytics показали, что различные стадии жизненного цикла существенно отличаются по уровню удержания, экономической ценности и степени влияния на устойчивость бизнеса.
#
### Key Findings Driving the Strategy
***Lifecycle Structure***

Последний snapshot клиентской базы имеет следующую структуру:

| Stage   | Customers |
| ------- | --------: |
| Active  |      1612 |
| At Risk |      1458 |
| Churn   |      1340 |
| Loyal   |       904 |
| New     |       404 |

Несмотря на относительно небольшой размер сегмента Loyal, именно он формирует ядро наиболее ценной клиентской аудитории.
#
***Revenue Concentration***

Распределение выручки между lifecycle-сегментами крайне неравномерно.

| Stage   | Revenue Share |
| ------- | ------------: |
| Loyal   |         55.4% |
| Active  |         22.8% |
| At Risk |         17.0% |
| New     |          4.8% |


Менее 16% клиентов генерируют более половины всей выручки компании. Это указывает на высокую зависимость бизнеса от удержания ядра постоянных покупателей.
#
***Lifecycle Stability***

Дополнительный анализ показал относительно стабильную структуру клиентской базы.

| Metric                    | Latest Value |
| ------------------------- | -----------: |
| Lifecycle Stability Index |        67.7% |
| Healthy Stability Index   |        71.0% |

Наиболее устойчивыми остаются клиенты из здоровых сегментов Active и Loyal.
#
### Strategic Recommendation 1
*Увеличить конверсию Active → Loyal*

Переход клиентов из Active в Loyal является наиболее важным направлением долгосрочного роста.
Именно в этом сегменте сосредоточен основной потенциал увеличения Customer Lifetime Value. Для ускорения перехода рекомендуется:
- программы лояльности;
- накопительные бонусы;
- персонализированные предложения;
- стимулирование повторных покупок.

**Expected Business Effect**  
- рост доли Loyal-клиентов;
- увеличение LTV;
- повышение устойчивости выручки.
#
### Strategic Recommendation 2
*Рассматривать Healthy Stability как основной KPI портфолио*

Healthy Stability Index показал высокий уровень устойчивости клиентов внутри здоровых lifecycle-состояний.
Данная метрика должна использоваться как один из ключевых индикаторов качества клиентского портфеля.
Снижение Healthy Stability Index может выступать ранним предупреждением ухудшения retention-модели еще до заметного роста churn.

**Expected Business Effect**
- раннее обнаружение деградации клиентской базы;
- повышение качества мониторинга customer health;
- более быстрое принятие управленческих решений.
#
### Strategic Recommendation 3
*Снизить Active → At Risk миграцию*

Результаты transition analytics показали, что именно Active-сегмент является ключевой точкой принятия решения клиентом о дальнейшем развитии жизненного цикла.
Переходы из Active в At Risk выступают одним из основных механизмов формирования будущего churn. Для сокращения данного потока рекомендуется:
- мониторинг ранних поведенческих сигналов;  
- триггерные коммуникации;  
- автоматизированные retention-сценарии;  
- сегментация по динамике активности.  

**Expected Business Effect**
- снижение будущего churn;
- повышение retention;
- стабилизация клиентской базы.
#
### Strategic Recommendation 4
*Защитить сегмент Loyal*

Loyal-клиенты являются основным источником выручки компании.
При этом даже незначительное ухудшение удержания данного сегмента способно привести к существенным финансовым потерям. Поэтому для Loyal-аудитории рекомендуется использовать отдельную стратегию:   
- VIP-коммуникации;  
- программы долгосрочной лояльности;  
- персонализированные предложения;  
- специальные механики удержания.  

**Expected Business Effect**
- защита 55% выручки компании;
- повышение Revenue Retention;
- снижение концентрации revenue risk.
#
### Strategic Implications

Результаты lifecycle analytics показывают, что долгосрочный рост бизнеса определяется не только привлечением новых клиентов, но прежде всего способностью компании переводить пользователей в устойчивые состояния жизненного цикла и удерживать ядро наиболее ценной аудитории.
Приоритетом стратегии должно стать увеличение доли клиентов в сегменте Loyal, сокращение миграции в At Risk и постоянный мониторинг устойчивости здоровой части клиентской базы.

## 13.3 Revenue Protection Strategy
### Strategic Objective

Основной целью стратегии защиты выручки является снижение концентрации финансовых рисков внутри клиентского портфеля и сохранение наиболее ценных источников дохода компании.

Результаты предыдущих разделов показали, что структура выручки аптечной сети является высококонцентрированной. Значительная часть бизнеса зависит от относительно небольшой группы клиентов, тогда как ухудшение поведенческих метрик начинается значительно раньше фактического ухода пользователей.

Поэтому управление выручкой должно быть направлено не только на предотвращение полного churn, но и на раннее выявление признаков снижения клиентской ценности, сокращения потребления и роста revenue-at-risk.
#
### Key Findings Driving the Strategy
***Revenue Concentration***

Анализ структуры выручки показал выраженную концентрацию бизнеса вокруг ядра наиболее ценных клиентов.

| Lifecycle Stage | Revenue Share |
| --------------- | ------------: |
| Loyal           |         55.4% |
| Active          |         22.8% |
| At Risk         |         17.0% |
| New             |          4.8% |


Несмотря на то что сегмент Loyal составляет менее 16% клиентской базы, именно он генерирует более половины совокупной выручки компании.
Это означает, что устойчивость бизнеса в значительной степени зависит от способности удерживать наиболее ценных клиентов.
#
***Revenue-at-Risk***

Интеграция прогнозных вероятностей оттока с финансовыми показателями позволила оценить объем выручки, находящейся под угрозой потери.

| Lifecycle Stage | Revenue-at-Risk |
| --------------- | --------------: |
| Active          |           1.39M |
| At Risk         |           1.35M |
| Loyal           |           1.02M |
| New             |           0.25M |


Наиболее важным результатом является то, что максимальный объем revenue-at-risk сосредоточен не только среди клиентов At Risk, но и внутри сегмента Active.
Таким образом, значительная часть будущих финансовых потерь формируется еще до того, как пользователь переходит в критическое состояние жизненного цикла.
#
***Revenue Dynamics***

Анализ Revenue Dynamics выявил ухудшение качества клиентской базы.
К концу периода наблюдения отрицательные изменения внутри удержанной базы начинают систематически превышать положительные.

| Period    | Expansion Revenue | Contraction Revenue |
| --------- | ----------------: | ------------------: |
| Dec → Jan |             1.06M |               0.58M |
| Jan → Feb |             1.06M |               0.84M |
| Feb → Mar |             1.23M |               1.15M |
| Mar → Apr |             0.93M |               1.34M |
| Apr → May |             1.06M |               1.49M |

Данный результат указывает на постепенное ухудшение customer health еще до возникновения полного churn.
#
### Strategic Recommendation 1
*Защитить ценную клиентскую базу*

Поскольку более половины выручки формируется Loyal-сегментом, данная группа должна рассматриваться как стратегический актив бизнеса. Для снижения риска потери наиболее ценных клиентов рекомендуется внедрить специализированную программу удержания, включающую:  
- персонализированные предложения;  
- VIP-программы лояльности;  
- индивидуальные механики удержания;  
- специальные сервисные преимущества;  
- превентивный мониторинг признаков деградации поведения.  

**Expected Business Effect**
- защита ключевого источника выручки;
- повышение Revenue Retention;
- снижение концентрации финансовых рисков.
#
### Strategic Recommendation 2
*Внедрить мониторинг Revenue-at-Risk*

Revenue-at-Risk должен стать регулярным управленческим показателем наряду с традиционными метриками продаж. Рекомендуется внедрить ежемесячный мониторинг:  
- Revenue-at-Risk by Lifecycle Stage;   
- Revenue-at-Risk by Customer Segment;  
- Revenue-at-Risk Trend;  
- Top Revenue-at-Risk Customers.  

Особое внимание следует уделять сегментам Active и At Risk, которые формируют максимальный объем потенциальных потерь.

**Expected Business Effect** 
- раннее обнаружение финансовых рисков;
- повышение точности планирования retention-бюджета;
- улучшение качества CRM-приоритизации.
#
### Strategic Recommendation 3
*Внедрить мониторинг баланса Expansion-Contraction*

Полный churn является лишь финальной стадией деградации клиентского поведения.
Значительно раньше начинают проявляться признаки сокращения потребления внутри удержанной базы.
Поэтому рекомендуется использовать показатель Expansion vs Contraction как один из ключевых индикаторов customer health.
Рост доли contraction должен рассматриваться как ранний сигнал ухудшения качества клиентского портфеля.

**Expected Business Effect**
- выявление проблем до роста churn;
- повышение скорости управленческой реакции;
- снижение скрытых потерь выручки.
#
### Strategic Recommendation 4
*Сместить фокус с Revenue Loss на Revenue Preservation*

Традиционный подход концентрируется на компенсации уже произошедших потерь.
Результаты анализа показывают необходимость перехода к модели превентивного управления выручкой. Основное внимание должно быть сосредоточено на:  
- предотвращении деградации ценных клиентов;  
- поддержании высокого Revenue Retention;   
- сохранении стабильности ядра клиентской базы;   
- снижении вероятности перехода клиентов в стадии риска.  

**Expected Business Effect**
- повышение устойчивости доходов;
- сокращение волатильности выручки;

# 14. Limitations & Future Work
## 14.1 Limitations
## 14.1.1 Methodological Limitations

Несмотря на высокую практическую применимость разработанного аналитического framework, результаты исследования следует интерпретировать с учетом ряда методологических ограничений.
#
### Observation Window

Анализ охватывает период с июля 2021 года по июнь 2022 года.

Данный горизонт наблюдения достаточен для исследования жизненного цикла клиентов, оценки retention dynamics и построения предиктивных моделей оттока. Однако более длительный период наблюдения позволил бы надежнее оценивать долгосрочные поведенческие паттерны и устойчивость выявленных закономерностей на поздних стадиях жизненного цикла клиентов.
#
### Cohort Maturity

Когортный анализ подвержен естественным ограничениям зрелости когорт.

По мере увеличения возраста когорт количество доступных наблюдений постепенно сокращается, что приводит к росту статистической неопределенности долгосрочных оценок retention и customer value.

В рамках проекта для повышения надежности результатов из расчета средних retention-кривых были исключены незрелые когорты. Тем не менее показатели за пределами шестого месяца жизненного цикла следует интерпретировать с осторожностью, поскольку в их расчете участвует ограниченное количество когорт.
#
### Snapshot-Based Lifecycle Segmentation

Lifecycle-модель построена на основе snapshot-подхода, в рамках которого пользователи классифицируются по дискретным стадиям жизненного цикла в отдельные моменты времени.

Подобный подход обеспечивает высокую интерпретируемость результатов и удобство практического применения, однако упрощает непрерывную природу пользовательского поведения. В реальности процесс деградации или восстановления активности клиентов может происходить более плавно, чем отражено в рамках фиксированных lifecycle-состояний.
#
### Recovery Modeling Scope

Модель оценки вероятности восстановления клиентов использовалась как exploratory ranking framework для анализа потенциала восстановления внутри сегмента At Risk.

Вероятности восстановления рассчитывались на тех же наблюдениях, на которых выполнялось обучение модели (in-sample scoring), и не рассматривались как самостоятельная готовая к внедрению в производство система прогнозирования (production-ready predictive system).

Поэтому полученные значения Recovery Probability следует интерпретировать прежде всего как относительные показатели потенциальной реактивируемости клиентов, используемые для приоритезации удержания, а не как точные прогнозы вероятности возврата к покупкам.

Основная задача модели заключалась в ранжировании клиентов по ожидаемому потенциалу восстановления и поддержке принятия решений о распределении retention-инвестиций.
#
### Predictive Revenue Estimation

Показатели Expected Future Revenue, Expected Revenue Loss и Retention Priority Score являются модельными оценками и не представляют собой точные значения будущей выручки.

Расчеты основаны на историческом поведении клиентов, вероятности оттока и вероятности восстановления активности, полученных в рамках предиктивного моделирования. Фактические результаты могут отличаться от прогнозируемых значений под воздействием факторов, не представленных в используемых данных, включая изменения рыночной ситуации, потребительского поведения, конкурентной среды и ценовой политики компании.

Дополнительно следует учитывать особенности построения показателя Expected Future Revenue. В рамках проекта прогнозируемая выручка рассчитывается на основе текущего значения Monetary и исторической динамики Monetary Growth.

При этом показатель Monetary представляет собой накопленную выручку клиента за последние шесть месяцев относительно даты snapshot, а Monetary Growth рассчитывается как отношение значений Monetary в двух последовательных snapshot-периодах. Поскольку соседние шестимесячные окна наблюдения пересекаются между собой, данный показатель отражает изменение долгосрочного покупательского тренда клиента, а не классический темп роста выручки между отдельными календарными месяцами. Monetary Growth отражает изменение клиентской ценности между последовательными lifecycle snapshots и используется как индикатор направления поведенческого тренда клиента.

В результате Expected Future Revenue следует интерпретировать как оценку относительной будущей ценности клиента, основанную на исторической динамике потребления, а не как точный финансовый прогноз будущих денежных потоков.

Аналогично показатель Expected Revenue Loss отражает ожидаемый объем выручки, находящейся под риском потери с учетом вероятности оттока клиента, и должен рассматриваться как инструмент оценки относительного бизнес-риска различных сегментов клиентской базы.

По этой причине показатели Expected Future Revenue и Expected Revenue Loss следует использовать прежде всего для сравнительного анализа клиентов и сегментов, а не для финансового планирования или прогнозирования фактической выручки.

Показатель Retention Priority Score объединяет вероятность оттока, ожидаемую будущую ценность клиента и вероятность восстановления активности. Его основная задача заключается не в прогнозировании денежных потоков, а в ранжировании клиентов по ожидаемой эффективности retention-интервенций и поддержке принятия решений по распределению retention-бюджета.

Таким образом, все три показателя следует интерпретировать как элементы системы приоритизации и оценки относительной бизнес-ценности клиентов, а не как точные прогнозы будущих финансовых результатов.
#
### Absence of Causal Validation

Проект основан на анализе наблюдательных данных и не включает результаты контролируемых экспериментов или A/B-тестирования retention-интервенций.

Следовательно, выявленные взаимосвязи отражают статистические закономерности пользовательского поведения и не позволяют делать строгие причинно-следственные выводы относительно эффективности конкретных маркетинговых воздействий.

Полученные результаты следует рассматривать как основу для дальнейшей проверки гипотез в рамках экспериментальных CRM-программ.

## 14.1.2 Data Limitations

Результаты исследования также ограничены составом доступных данных и уровнем детализации пользовательской информации.

В основе анализа лежат транзакционные данные клиентов, отражающие историю покупок, частоту взаимодействия с компанией и объем сформированной выручки.
Такой подход обеспечивает надежную основу для исследования customer lifecycle и retention динамик, однако не позволяет учитывать дополнительные факторы пользовательского поведения, которые могут оказывать влияние на вероятность удержания или оттока клиентов.
#

В доступных данных отсутствует информация о маркетинговых коммуникациях и CRM-взаимодействиях с клиентами.

В результате модели не учитывают влияние email-кампаний, push-уведомлений, программ лояльности и других retention-механик на дальнейшее поведение пользователей.

Это ограничивает возможность количественной оценки эффективности отдельных маркетинговых воздействий и построения моделей инкрементального эффекта CRM-активностей.
#
### Absence of Acquisition Channel Information

Проект не содержит информации об источниках привлечения клиентов.

Поэтому различия в качестве пользовательских когорт анализируются исключительно через поведенческие метрики после первой покупки без учета особенностей каналов acquisition.

Наличие данных о происхождении клиентов позволило бы дополнительно исследовать связь между источниками привлечения, долгосрочным удержанием и формированием customer lifetime value.
#
### Absence of Product-Level Information

В анализируемом наборе данных отсутствует информация о товарных категориях и структуре потребления внутри покупательской корзины.

По этой причине модели рассматривают клиентов как единые поведенческие объекты и не позволяют оценивать влияние продуктовых предпочтений на вероятность удержания, реактивации и долгосрочную ценность пользователей.

Интеграция категорийного анализа могла бы существенно расширить возможности персонализации retention-стратегий и формирования более точных клиентских сегментов.

## 14.2.1 Survival Analysis
### Бизнес-мотивация

В текущем проекте задача оттока сформулирована как бинарная классификация:  
- клиент перейдет в риск/отток;  
- клиент не перейдет в риск/отток.  

Такой подход позволяет оценивать вероятность нежелательного события, однако не отвечает на вопрос:   
> когда именно клиент может быть потерян?

Для CRM-команд этот вопрос часто имеет не меньшую ценность, чем сама вероятность оттока.

Например:
- два клиента могут иметь одинаковый churn-risk;  
- при этом один клиент может уйти через неделю;  
- другой — через несколько месяцев.

При одинаковой вероятности риска приоритет коммуникаций будет различаться.
#
### Возможное решение

Для оценки времени до оттока может использоваться Survival Analysis. Вместо прогнозирования класса модель оценивает:  
- вероятность сохранения активности клиента во времени;  
- ожидаемое время до оттока;  
- изменение риска ухода на различных этапах жизненного цикла.  
- Потенциальная бизнес-ценность
#  
### Потенциальная бизнес-ценность

Интеграция survival-моделей позволит:  
- прогнозировать время до потери клиента;  
- выявлять периоды максимального риска;  
- оптимизировать момент запуска retention-коммуникаций;  
- повысить эффективность CRM-кампаний за счет более точного тайминга взаимодействия.  

Таким образом, компания сможет перейти от оценки вероятности ухода к прогнозированию полного жизненного цикла клиента.

## 14.2.2 Customer Lifetime Value Forecasting
### Бизнес-мотивация

В рамках проекта используется показатель Expected Future Revenue, который позволяет оценивать потенциальную ценность клиентов для целей retention prioritization.

Однако данный показатель отражает ожидаемую будущую выручку в рамках текущего аналитического подхода и не является полноценным долгосрочным прогнозом Customer Lifetime Value.

Для стратегического управления клиентской базой компании важно понимать:  
- какую совокупную выручку клиент способен принести в будущем;  
- какие сегменты обладают максимальным долгосрочным потенциалом;  
- насколько различается экономическая ценность разных типов пользователей.  
#
### Возможное решение

Следующим этапом развития системы может стать построение моделей Customer Lifetime Value Forecasting. Такие модели могут учитывать:
- историю покупок;
- динамику потребления;
- churn-risk;
- lifecycle stage;
- поведенческие тренды клиента.

Результатом является прогноз совокупной будущей ценности пользователя на протяжении всего оставшегося жизненного цикла.
#
### Потенциальная бизнес-ценность

CLV-модели позволяют:
- оптимизировать распределение маркетингового бюджета;
- определять максимально ценных клиентов;
- оценивать долгосрочную эффективность retention-стратегий;
- принимать решения на основе ожидаемой полной ценности клиента, а не только текущей выручки.

В сочетании с churn-моделями CLV-прогнозирование формирует основу для полноценного customer portfolio management.

## 14.2.3 Uplift Modeling
### Бизнес-мотивация

Текущий Retention Priority Framework позволяет определить:  
- кого компания может потерять;  
- сколько выручки находится под угрозой;  
- каких клиентов потенциально возможно вернуть.  

Однако даже высокая вероятность восстановления активности не означает, что именно CRM-интервенция станет причиной этого восстановления.
Некоторые клиенты могут вернуться самостоятельно без какого-либо воздействия.
Другие пользователи могут не реагировать на коммуникации независимо от уровня инвестиций.
Поэтому следующий уровень зрелости retention-аналитики связан с оценкой причинно-следственного эффекта маркетинговых воздействий.
#
### Возможное решение

Для решения данной задачи может использоваться Uplift Modeling. В отличие от классических predictive-моделей uplift-подход оценивает:    
- не вероятность события,   
- а дополнительную вероятность события, вызванную конкретной интервенцией.  

То есть модель отвечает на вопрос:
> Насколько изменится вероятность сохранения клиента именно благодаря CRM-коммуникации?
#
### Потенциальная бизнес-ценность

Использование uplift-моделей позволяет:
- исключать клиентов, которые вернутся самостоятельно;
- исключать клиентов, не реагирующих на коммуникации;
- концентрировать ресурсы на действительно управляемой аудитории;
- повышать ROI retention-кампаний;
- оптимизировать маркетинговые расходы.

В результате компания сможет перейти от прогнозирования риска к прогнозированию эффекта управленческих действий.
#
### *Иллюстративный пример*
#### *Концептуальная матрица retention-инвестиции* 

|                                  | **Низкая Expected Future Revenue** | **Высокая Expected Future Revenue** |
| -------------------------------- | ---------------------------------- | ----------------------------------- |
| **Низкая Recovery Probability**  | **Low Priority**                   | **Selective Retention**             |
| **Высокая Recovery Probability** | **Test Campaigns**                 | **Primary Target Segment**          |

*Иллюстративный пример концептуальной матрицы распределения retention-инвестиций на основе ожидаемой будущей ценности клиента (Expected Future Revenue) и вероятности успешной реактивации (Recovery Probability).*

*В качестве концептуальной архитектуры будущей системы можно использовать матрицу распределения retention-инвестиций.*

***Primary Target Segment***  
*Высокая Expected Future Revenue + высокая Recovery Probability.*    
*Наиболее ценные и наиболее "спасаемые" клиенты.*    
*Именно на данный сегмент должен направляться основной объем retention-бюджета.*  

***Selective Retention***  
*Высокая Expected Future Revenue + низкая Recovery Probability.*  
*Клиенты обладают высокой ценностью, однако вероятность успешного восстановления активности ограничена.*  
*Для данного сегмента целесообразно использовать выборочные персонализированные кампании и контролировать их ROI.*  

***Test Campaigns***  
*Низкая Expected Future Revenue + высокая Recovery Probability.*  
*Клиенты легко поддаются реактивации, однако обладают ограниченной экономической ценностью.*  
*Сегмент может использоваться для недорогих автоматизированных CRM-экспериментов.*  

***Low Priority***  
*Низкая Expected Future Revenue + низкая Recovery Probability.*  
*Клиенты одновременно обладают низкой ценностью и низким потенциалом восстановления.*  
*Retention-инвестиции для данного сегмента, как правило, обладают минимальной ожидаемой отдачей.*  

*Предложенная матрица демонстрирует возможный переход от текущей системы risk-based retention к более зрелому подходу, основанному на оценке инкрементального эффекта CRM-интервенций.*  

## 14.2.4 Causal Retention Analytics
### Бизнес-мотивация

В рамках текущего проекта были выявлены ключевые факторы оттока клиентов, рассчитаны churn probabilities и сформирована система приоритизации retention-инвестиций.

Однако построенные модели позволяют выявлять статистические закономерности, но не позволяют однозначно ответить на вопрос:  
> какие факторы действительно вызывают изменения клиентского поведения?

Например, анализ показал, что снижение Frequency и ухудшение Monetary Dynamics тесно связаны с ростом churn-risk. Тем не менее наличие статистической связи еще не означает наличие причинно-следственного эффекта.

Для принятия управленческих решений компании важно понимать:  
- какие изменения поведения действительно приводят к оттоку;  
- какие CRM-интервенции способны изменить траекторию клиента;  
- какие факторы являются причиной, а какие лишь индикаторами деградации.  
#
### Возможное решение

Следующим этапом развития аналитической системы может стать внедрение методов Causal Analytics.
В отличие от классических predictive-моделей причинно-следственный анализ позволяет оценивать влияние конкретных факторов и действий на будущие результаты клиента. Потенциальные направления анализа:  
- оценка влияния изменения частоты покупок на вероятность оттока;  
- анализ эффекта повторных коммуникаций;  
- оценка влияния программ лояльности на удержание клиентов;  
- измерение долгосрочного эффекта retention-кампаний.  

Для решения подобных задач могут использоваться:  
- propensity score matching;   
- causal forests;  
- difference-in-differences;  
- treatment effect estimation.  
#
### Потенциальная бизнес-ценность

Использование причинно-следственного подхода позволит:  
- перейти от анализа корреляций к анализу причин;  
- повысить эффективность retention-инвестиций;  
- измерять реальный эффект CRM-активностей;  
- снижать риск принятия решений на основе ложных зависимостей;  
- формировать более обоснованные стратегии удержания клиентов.  

В результате компания сможет понимать не только кто находится в зоне риска, но и какие действия способны изменить будущий исход.
#

#### *Figure 71. Концептуальная модель перехода от корреляционного анализа факторов риска к оценке причинного влияния retention-интервенци*

## 14.2.5 Next-Best-Action Recommendation System
### Бизнес-мотивация

Текущая аналитическая система позволяет определить:  
- вероятность оттока клиента;  
- ожидаемую будущую ценность;  
- потенциальный объем revenue-at-risk;  
- приоритет удержания. 

Однако после определения приоритетных клиентов остается открытым следующий вопрос:   
> какое именно действие следует предпринять для каждого пользователя?

На практике одинаковые retention-коммуникации редко обеспечивают максимальный результат для всей клиентской базы.

Различные группы клиентов могут требовать разных стратегий взаимодействия:
- персонализированных предложений;
- напоминаний о повторной покупке;
- программ лояльности;
- стимулирующих скидок;
- реактивационных кампаний.
#
### Возможное решение

Следующим этапом развития CRM Intelligence Framework может стать построение системы Next-Best-Action Recommendation. Такая система объединяет:  
- lifecycle stage;  
- churn probability;  
- customer value;  
- recovery potential;  
- историю поведения клиента.  

На основе совокупности факторов каждому пользователю автоматически назначается наиболее подходящее действие.

Пример логики рекомендаций:  

| Клиентский профиль                   | Рекомендуемое действие   |
| ------------------------------------ | ------------------------ |
| Loyal + высокий revenue              | программа лояльности     |
| Active + ухудшение Frequency         | превентивное удержание   |
| At Risk + высокий recovery potential | реактивационная кампания |
| At Risk + низкая ценность            | минимальные коммуникации |
| Churn + высокий потенциал возврата   | win-back стратегия       |

#
### Потенциальная бизнес-ценность

Next-Best-Action Framework позволяет:  
- автоматизировать принятие CRM-решений;  
- повысить персонализацию коммуникаций;  
- оптимизировать использование retention-бюджета;  
- улучшить клиентский опыт;  
- увеличить возврат на маркетинговые инвестиции.  

Фактически система превращает результаты аналитики в конкретные управленческие рекомендации для каждого клиента.
